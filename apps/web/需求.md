# apps/web 功能需求文档

## 项目概述
企业知识库工作台前端（Next.js 16 + React 19 + Tailwind v4）。面向知识库治理的 Web 应用，提供文档批量入库、人工复核、文档库管理、检索验证、知识库集合管理等功能。

---

## 一、全局功能

### 1.1 应用外壳（AppShell）
- **左侧导航栏**
  - Logo + 产品名称 "Knowledge Workbench"
  - 导航项：批量入库、人工复核、文档库、检索验证、知识库集合
  - 当前页面高亮（背景色 + 文字色 + 右侧脉冲圆点）
  - 移动端：汉堡菜单触发抽屉式侧边栏，遮罩层关闭
  - 桌面端：可折叠/展开侧边栏（带动画 width 过渡）
- **顶部 Header**
  - 侧边栏切换按钮（汉堡/X 图标动画切换）
  - 后端服务健康状态指示灯（Admin / Workbench / Access / Retrieval / Ingestion）
  - "All Healthy" 状态徽标（绿点脉冲动画）
  - 知识库集合选择器（下拉选择当前操作的集合，从 /workbench/me 获取租户）
  - 设置入口按钮
- **内容区**
  - 页面切换时淡入 + 上移动画
  - 最大宽度 max-w-7xl，居中布局
- **底部 Footer（侧边栏底部）**
  - "Retrieval workbench" / "Non-answer-generation agent"

### 1.2 全局状态管理（Zustand）
- `currentCollectionId`: 当前选中的知识库集合 ID
- `demoToken`: 演示 JWT 令牌
- `demoApiKey`: 演示 API 密钥（访问服务用）
- `accessScope`: 权限范围（内部/外部，含部门、角色、用户、组、代理类型、客户、应用等字段）

### 1.3 全局交互
- **Toast 通知**：操作成功/失败提示（sonner）
- **错误边界**：捕获渲染错误，展示友好错误页
- **TooltipProvider**：全局 Tooltip 上下文
- **动画系统**：staggerContainer / staggerItem / overlayFade / slideInFromLeft 等预设动画
- **后端缺口（BackendGap）提示**：当 API 返回 501/未实现时，展示功能尚未就绪的提示卡片

### 1.4 API 客户端
- Workbench API 封装：支持超时控制、JWT Cookie 同步、401 自动登出
- 错误处理：区分 ApiClientError / BackendGapError / 网络错误
- 健康检查：/health/all 轮询（30s 间隔）

## 三、批量入库页（/upload）

### 3.1 上传区
- **拖拽上传**：支持拖拽文件到上传区（高亮边框 + 渐变背景动画）
- **点击上传**：点击区域唤起文件选择器，支持多选
- **支持格式**：PDF、DOCX、PPTX、XLSX、CSV
- **前置校验**：未选择集合或未配置权限范围时禁止上传，Toast 提示错误
- **并发控制**：最大 3 个并发上传，其余排队

### 3.2 文件状态追踪
- 每个文件卡片展示：文件类型图标、文件名、文件大小、状态徽标
- **状态流转**：排队 → 上传中 → 已上传 → 解析中 → 复核中 → 已批准/已发布 → 索引构建中
- 失败状态：展示错误信息，提供重试按钮
- 上传中/解析中/索引构建中：底部显示进度条（shimmer 动画）
- 卡片左侧状态色条（根据状态变色）
- 删除按钮：从列表移除

### 3.3 统计面板
- 当前批次统计卡片（5 个指标）：总数、处理中、已入库、待复核、失败
- 仅在存在文件时展示

### 3.4 最近任务列表
- 无限滚动加载（IntersectionObserver）
- 每页 20 条，按上传时间倒序
- 展示：文件名、集合 ID、状态徽标
- 加载更多指示器、总记录数提示
- 当前文件与任务状态联动：通过任务 API 轮询（5s/30s）更新文件状态

### 4.1 工单列表

### 4.2 筛选器

- **状态筛选**：全部、待复核、已批准、已拒绝、已退回
- 清除筛选按钮（仅在筛选非默认时展示）

### 4.3 空状态
- 无工单时展示 EmptyState 插图 + 描述

## 五、复核详情页（/review/[taskId] & /workbench/[ticketId]）

### 5.1 页面头部（Hero Section）
- 返回按钮（链接到复核队列）
- 状态徽标组：工单状态、失败码、下一步动作、文档关联来源
- 文档标题（回退链：文件名 → doc_id → ticket_id）
- 描述文案
- **四宫格指标卡片**：Findings 数、Risks（Critical/High）数、Draft edits 数、Warnings 数
- **元数据网格**：工单 ID、集合 ID、文档 ID、解析快照 ID

### 5.2 数据源降级提示
- 当 workspace 部分数据不可用时，展示 Alert 提示 degraded_parts

### 5.3 主内容区（三 Tabs）

#### Tab 1: Source（源文件预览）
- 文档查看器组件：
  - 支持 PDF（Canvas 渲染，分页 + 缩放）、图片、HTML（iframe）、纯文本
  - 工具栏：缩放（50%-200%）、复制解析文本、下载源文件
  - 分页导航：上一页/下一页、页码输入跳转
  - 锚定页按钮：根据 chunks 的 page_spans 生成快捷跳转按钮
  - 双 Tab：Source（原始文件）/ Parsed text（解析后文本）
  - 文本搜索高亮：支持从 Agent Review 点击 "Find in document" 自动搜索并高亮定位
  - 无源文件时展示 Alert 提示
- 解析上下文卡片：展示源文件名、后缀、解析器 ID、后端、生效策略
- 预览信号卡片：Chunks 数、Warnings 数、Edits 数、预览文本（max-h-52 滚动区域）

#### Tab 2: Draft edits（草稿编辑）
- 仅在有解析快照时可用
- ChunkEditorWorkbench 组件：
  - 搜索 chunks（按内容或 evidence_id）
  - 展示 chunk 列表：evidence_id Badge、section_path、页码范围
  - 展开/折叠长内容（>300 字符）
  - 编辑按钮：打开 ChunkEditModal
  - 高亮定位：从 Agent Review 点击 "Edit chunk" 自动滚动到对应 chunk
- 预发布模式：编辑提交到草稿层，不直接修改检索索引

#### Tab 3: Agent review（代理审核）
- AgentReviewPanel 组件：
  - 按严重程度排序（Critical → High → Medium → Low → Info）
  - 每条 Finding 卡片：严重程度 Badge、分类 Badge、置信度、问题摘要
  - 展开后展示：Source quote（引用块）、"Find in document" 按钮、"Edit chunk" 按钮、页码范围

### 5.4 右侧边栏（Decision Cockpit + Ticket Context + System & Diagnostics）

#### Decision Cockpit
- 当前状态展示（Badge 组）
- 若工单处于待复核状态且有权限：
  - 决策原因文本框（可选）
  - 三个操作按钮：Approve（通过）、Reject（驳回）、Return for revision（退回修改）
- 若已有决策：展示决策原因（或 "No additional reason"）

#### Ticket Context
- 工单 ID、集合 ID、文档 ID、解析快照 ID

#### System and Diagnostics
- 系统决策标签（System auto-approved/rejected/returned）
- 决策者、决策原因
- 失败码、失败阶段、下一步动作、降级部件

### 5.5 服务端聚合
- 页面数据通过 `workbenchApi.getWorkspaceDetail(ticketId)` 一次性获取聚合视图
- 包含：ticket、document、parse_snapshot、chunks、chunk_edits、agent_review、capabilities、degraded_parts

## 六、文档库页（/documents）

### 6.1 文档列表
- 列表展示所有文档投影（DocumentProjection），按 projection_updated_at 倒序，最多 200 条
- 每条文档卡片展示：
  - 复选框（管理员可批量选择）
  - 文件类型图标（PDF/DOC/PPT/Sheet，按扩展名匹配不同颜色）
  - 文件名（可点击跳转 `/documents/[docId]`）
  - Review 状态 Badge、Indexed Badge
  - 集合 ID、chunk 数、页数、完整度（Complete/Partial）、相对更新时间
  - 降级原因（如有）
  - 文档状态徽标（Active/Pending/Archived）
  - Stale 标记（红色 destructive Badge）
  - 右侧 Chevron 跳转按钮

### 6.2 筛选器（7 个）
- 关键词搜索（文件名或 doc_id 模糊匹配）
- 集合筛选
- 状态筛选（Active/Pending/Archived）
- 复核状态筛选（Pending/Approved/Rejected）
- 文件类型筛选（PDF/Document/Presentation/Spreadsheet）
- 新鲜度筛选（Stale only / Fresh only）
- 索引状态筛选（Indexed / Not indexed）
- 可见文档计数展示

### 6.3 批量操作（仅限管理员）
- 选择文档后展示批量操作栏：Archive（归档）、Retract（撤回）、Reindex（重建索引）
- 全选/取消全选当前可见文档
- 操作确认弹窗：输入原因、Reindex 需输入 index_profile_id
- 批量操作结果展示：总数量、成功数、失败数、每条结果的详细状态

### 6.4 管理员标识
- 拥有 knowledge_admin 或 platform_admin 角色的用户，展示 "Admin lifecycle enabled" Badge

## 七、文档详情页（/documents/[docId]）

### 7.1 页面结构
- 与复核详情页类似，但面向已发布文档的生命周期管理
- 头部展示：文档状态、Review 状态、Task 状态、STALE 标记、降级部件标记
- **四宫格指标**：Findings、Blocking（Critical/High）、Chunks、Draft edits
- **元数据网格**：Doc ID、Collection、Source File、Parse Snapshot

### 7.2 主内容区（三 Tabs）
- Source：同复核详情页的 DocumentViewer
- Chunks：同复核详情页的 ChunkEditorWorkbench（post-publish 模式）
- Agent：同复核详情页的 AgentReviewPanel

### 7.3 右侧边栏

#### Review Cockpit
- 若文档关联了复核工单：展示工单状态、决策、决策原因
- 待复核状态 + 有权限时：提供 Approve/Reject/Return 按钮 + 原因输入

#### Lifecycle Actions
- **Archive Document**：归档文档（需有 can_archive 权限）
- **Retract Document**：撤回文档（destructive 样式，需有 can_retract 权限）
- **Reindex Document**：重建索引（需有 can_reindex 权限）
- 无权限时展示提示文案

#### Diagnostics
- Task Status、Index Version、Next Action、Failure Stage
- 失败码 Alert 展示

### 7.4 生命周期操作弹窗
- 输入操作原因
- Reindex 需输入 index_profile_id
- Confirm / Cancel

## 八、检索验证页（/retrieval）

### 8.1 检索表单
- **查询输入框**：标准检索字段，Enter 触发检索
- **Token 预算输入**：数字输入，默认 2000
- **集合选择器**：下拉选择知识库集合（从 workbench 获取）
- **检索配置选择器**：下拉选择已发布的检索配置（retrieval profile）
- **调试级别选择**：无 / 基础 / 完整（Tooltip 解释各级含义）
- **检索按钮**：未选择集合或配置时禁用
- 未选集合/配置时展示 Alert 提示

### 8.2 检索结果
- 证据片段列表（evidence_items）
- 结果头部：标题 + 展开全部/收起全部按钮 + 计数 Badge（X 项 · 已用 Y Token）
- 每条证据卡片：
  - 序号 Badge（渐变背景）
  - 相关性分数（Score）+ 可视化进度条
  - 复制内容按钮（带 1.5s 成功状态反馈）
  - 展开/折叠按钮
  - 元数据行：doc_id、evidence_id、collection_id
  - 内容区（默认折叠 4 行）
  - 展开后展示：source_stage、why_selected、section_path

### 8.3 空结果
- 展示 EmptyState："无证据片段" + 提示检查查询/集合/配置

## 九、知识库集合页（/collections）

### 9.1 集合列表
- 卡片网格布局（1-3 列响应式）
- 每张卡片展示：
  - 文件夹图标 + 集合名称
  - 生命周期状态 Badge（active/其他）
  - 集合 ID（monospace，小号字）
  - 描述（2 行截断，无描述显示 "无描述"）
  - 租户 ID
  - "选择用于上传" 按钮：点击后设置当前集合并 Toast 成功提示
- 卡片悬停渐变覆盖效果

### 9.2 新建集合
- 弹窗表单：集合 ID、名称、描述
- 提交按钮：依赖租户 ID（从 /workbench/me 获取）
- 创建成功后：刷新列表、关闭弹窗、重置表单、Toast 成功
- 后端缺口时展示 BackendGap

### 9.3 加载与空状态
- 加载中：3 个骨架屏卡片
- 错误：Alert 展示错误信息
- 无集合：EmptyState + 创建按钮

## 十二、技术约束

- **框架**：Next.js 16（App Router），React 19
- **样式**：Tailwind CSS v4，CSS 变量主题系统
- **字体**：Geist Sans + Geist Mono（Google Fonts）
- **状态管理**：Zustand（客户端全局状态）
- **数据获取**：TanStack Query v5（React Query）
- **动画**：Framer Motion（页面过渡、列表 stagger、tab 切换、侧边栏展开）
- **通知**：Sonner（Toast）
- **图标**：Lucide React
- **PDF 预览**：pdfjs-dist（动态导入 webpack.mjs 入口）
- **服务端**：无 SSR 数据获取（所有数据在客户端通过 React Query 获取），页面标记 "use client"

## 十三、待办功能模块（产品经理视角补充）

> 以下功能基于企业级知识库产品的治理层、分析层、协作层需求梳理，尚未进入开发排期。

### 13.1 数据仪表盘（Dashboard）
- **总览指标卡片**：今日上传量、待复核队列深度、检索调用次数、文档健康度（Stale 比例）
- **趋势图表**：7/30 天入库趋势、审核通过率变化、Top 检索查询词云
- **快捷入口区**：最近处理的 5 条工单、即将过期的文档提醒
- **状态**：当前首页（`/`）直接重定向到 `/upload`，缺少"进门先看全局"的入口页面
- **优先级建议**：高（改善首屏体验，便于运营决策）

### 13.2 通知中心
- **消息类型**：
  - 工单状态变更（被驳回/通过/退回）
  - Chunk 编辑被他人覆盖冲突提示
  - 集合配额/存储空间预警
  - 系统维护公告
- **消息列表**：未读红点标记、按时间倒序、点击跳转对应业务页面
- **推送配置**：站内通知开关、按事件类型订阅/退订
- **位置**：顶部 Header 右侧铃铛图标，下拉面板展示
- **优先级建议**：中（提升协作效率）

### 13.3 检索配置管理（Retrieval Profile Admin）
- **配置 CRUD**：创建/编辑/删除检索策略配置
  - 可调参数：重排序模型、TopK、相似度阈值、Token 预算上限、过滤规则（metadata 过滤）
  - 发布/草稿状态切换
- **A/B 测试面板**：同一查询用两套配置并行检索，并排对比结果差异
- **版本历史**：配置变更记录、回滚到历史版本
- **页面路由建议**：`/retrieval/profiles`
- **优先级建议**：高（当前检索验证页只"使用"配置，无管理入口，产品闭环不完整）

### 13.4 解析策略管理（Parse Policy Admin）
- **解析器配置**：按文件类型绑定不同解析器（PDF→DeepDoc、HTML→html-parser、CSV→结构化提取等）
- **预处理规则开关**：
  - OCR 文字识别启用/禁用
  - 表格提取策略（保留结构/转文本/忽略）
  - 页眉页脚自动过滤
  - 分块策略（固定长度/语义段落/重叠大小）
- **预览比对**：上传同一文件，用两套策略解析后并排展示 chunk 差异
- **页面路由建议**：`/settings/parse-policies`
- **优先级建议**：中（影响解析质量，但当前有默认策略可跑通）

### 13.5 审计日志（Audit Log）
- **记录范围**：谁、何时、对哪个文档/集合做了什么操作
  - 操作类型：上传、批准、拒绝、退回、编辑 chunk、归档、撤回、重建索引、删除
  - 涉及字段：操作人 ID、IP、时间戳、操作对象、变更前/后快照（可选）
- **筛选能力**：按操作人、操作类型、时间范围、集合、文档 ID
- **导出能力**：CSV/Excel 导出，用于合规审查
- **展示约束**：前端只读，不可编辑/删除；由后端保证不可篡改
- **页面路由建议**：`/settings/audit-log`
- **优先级建议**：高（企业合规刚需，安全审计必备）

### 13.6 工单协作与评论
- **评论系统**：复核工单项下支持多轮评论
  - 富文本/纯文本输入、@同事提及（下拉选人）
  - 评论附件（截图、补充文档）
  - 评论时间轴展示
- **指派机制**：管理员可将待复核工单指派给特定审核员
  - 被指派人收到通知
  - 工单列表增加"我的待办"筛选
- **处理记录时间轴**：单条工单的完整生命周期可视化
  - 上传 → 解析 → Agent 审核 → 人工复核 → 发布/驳回
  - 每个节点展示耗时、处理人、决策原因
- **优先级建议**：中（多人协作场景的核心能力）

### 13.7 API 密钥与访问管理（正式版）
- **密钥生命周期管理**：
  - 用户自主创建 API 密钥（命名、过期时间设置、可选永不过期）
  - 查看已创建密钥列表（只展示前缀，完整密钥仅创建时可见）
  - 吊销/删除密钥
- **权限绑定**：密钥与特定集合/操作权限绑定（细粒度授权）
- **用量统计**：每个密钥的调用次数、Token 消耗、QPS 峰值、最近调用时间
- **密钥轮换向导**：生成新密钥 → 迁移调用方 → 吊销旧密钥（防中断引导）
- **页面路由建议**：`/settings/api-keys`
- **优先级建议**：高（当前设置页只有"演示 APIKey"输入框，不具备产品化能力）

### 13.8 回收站（Trash）
- **触发条件**：用户归档或删除文档时，文档进入回收站（软删除）
- **功能**：
  - 列表展示回收站文档（原集合、删除时间、删除人）
  - 恢复文档到原集合（需校验集合是否仍存在）
  - 彻底删除（二次确认弹窗，destructive 样式）
  - 批量恢复/彻底删除
- **自动清理策略**：30 天后自动物理删除（后台任务），前端展示倒计时
- **页面路由建议**：`/documents/trash`
- **优先级建议**：中（防误删，数据安全兜底）

### 13.9 帮助中心与 Onboarding
- **新用户引导（Product Tour）**：
  - 首次登录时触发 Spotlight 式分步引导
  - 覆盖：上传区、集合选择器、复核队列、决策按钮
  - 支持"跳过"和"稍后提醒我"
- **页面内帮助**：每个页面 Header 右侧 "?" 按钮，弹出该页面功能说明浮层
- **快捷键面板**：全局快捷键一览
  - `Cmd/Ctrl + K`：全局搜索/跳转
  - `/`：聚焦当前页面搜索
  - `?`：打开快捷键面板
- **反馈入口**：右下角悬浮反馈按钮，提交问题/建议（对接工单系统或邮件）
- **优先级建议**：低（体验优化，非阻塞功能）

### 13.10 全局搜索（Command Palette）
- **触发方式**：`Cmd/Ctrl + K` 或点击顶部搜索框

- **展示**：模态浮层，居中，暗色半透明遮罩
- **优先级建议**：中（提升导航效率，适合文档/工单数量增长后的场景）

## 十四、现有页面功能缺口分析（产品经理视角）

> 逐页审视已开发页面，从操作效率、信息完整性、协作能力、容错兜底四个维度梳理当前缺失的功能。

### 14.1 首页（/）
- **当前状态**：仅做服务端重定向到 `/upload`，无实质内容
- **缺失功能**：
  - 数据仪表盘入口（见 13.1）
  - 快捷操作区（一键进入最近任务、待复核队列、常用集合）
  - 系统公告/运营 Banner（版本更新、维护通知）
- **产品影响**：用户每次进入产品都直接跳进上传页，缺少"总览全局"的心理锚点，不利于运营触达

### 14.2 批量入库页（/upload）
- **当前能力**：拖拽上传、状态追踪、统计面板、最近任务无限滚动
- **缺失功能**：
  - **文件搜索过滤**：当前批次文件多了之后无法按文件名/状态快速定位
  - **排序能力**：按上传时间/文件名/文件大小/状态排序
  - **批量操作**：多选文件后批量删除、批量重试失败项
  - **任务取消**：上传中/解析中的任务应支持取消（释放后端资源）
  - **上传速度/ETA**：展示实时上传速度和预计剩余时间，减少用户焦虑
  - **文件大小预警**：超过阈值（如 50MB）时前置提示"大文件解析可能耗时较长"
  - **上传模板下载**：如果业务有标准化要求（如 CSV 字段规范），提供模板下载入口
  - **断点续传/分片上传**：网络中断后自动恢复，而非从头重来
  - **上传历史持久化**：刷新页面后当前批次文件列表丢失（仅保留最近任务）
- **产品影响**：文件一多就难管，大文件失败成本高，缺少掌控感

### 14.3 人工复核队列页（/review）
- **当前能力**：工单列表、集合/状态双筛选、相对时间、清除筛选
- **缺失功能**：
  - **搜索能力**：按工单 ID / 文件名 / 文档 ID 关键词搜索
  - **排序能力**：按更新时间 / 创建时间 / 文件名排序（当前固定按时间倒序）
  - **分页/虚拟滚动**：当前一次性加载 100 条，工单量大时首屏卡顿
  - **优先级标记**：区分 P0/P1/P2 优先级，高优先级置顶或高亮
  - **分配人维度**："我的待办" vs "全部" vs "某同事的待办"切换
  - **批量审批**：勾选多条工单一键 Approve / Reject（需确认弹窗）
  - **队列统计**：顶部展示"待复核 12 / 今日已处理 8 / 平均处理时长 15min"
  - **卡片快捷预览**：Hover 展开浮层展示 Agent Review 摘要（不用点进去）
  - **自动刷新开关**：手动刷新 / 自动轮询（5s）切换
  - **空队列占位**：当前仅 EmptyState，可补充"恭喜，队列已清空"正向反馈+引导语
- **产品影响**：审核员每天面对几十上百条工单，没有搜索和批量操作效率极低

### 14.4 复核详情页（/review/[taskId] & /workbench/[ticketId]）
- **当前能力**：三 Tabs（源文件/草稿/Agent审核）、决策驾驶舱、系统诊断
- **缺失功能**：
  - **工单评论/讨论区**：团队成员围绕该工单留言讨论（富文本、@提及、附件）
  - **工单转让**：审核员可将工单转给他人处理（附带转让原因）
  - **完整时间轴**：上传 → 解析完成 → Agent 审核完成 → 人工复核 → 决策提交 → 发布，每个节点带耗时
  - **源文件 vs 解析结果 并排对比**：左右分栏对比原始内容和解析后的 chunk，直观发现解析偏差
  - **审核报告导出**：一键导出 PDF/邮件，包含文档信息、Agent Findings、决策结论
  - **审核意见模板**：下拉选择常见拒绝原因（"格式不规范"/"敏感信息"/"内容重复"），减少重复输入
  - **相似工单推荐**："该文档与工单 #xxx 内容相似，建议参考之前的处理结论"
  - **审核计时器**：展示"该工单已等待审核 2h 30min"，督促 SLA
  - **并发编辑冲突**：多人同时打开同一工单审核时，提示"某某正在处理此工单"
  - **工作台差异化**：当前 `/review/[taskId]` 和 `/workbench/[ticketId]` 复用同一组件，缺少工作台特有的"草稿自动保存"、"预发布预览"能力
- **产品影响**：审核是核心链路，缺少协作和时间轴导致责任不清、重复劳动

### 14.5 文档库页（/documents）
- **当前能力**：文档列表、7 维筛选、批量操作（管理员）、文件类型图标
- **缺失功能**：
  - **排序能力**：当前固定按 projection_updated_at 倒序，应支持按文件名 / 页数 / chunk 数 / 状态排序
  - **视图切换**：列表视图（当前）vs 卡片/网格视图（展示文档缩略图/预览）
  - **快速预览**：点击预览按钮在浮层中查看文档摘要和关键 chunk，不用跳转详情页
  - **文档标签系统**：给文档打自定义标签（"重要"/"待更新"/"已过期"），按标签筛选
  - **收藏/置顶**：用户可将常用文档加入收藏，在列表顶部展示
  - **最近访问**：展示"最近查看的 5 个文档"快捷入口
  - **文档质量评分**：展示文档完整度/解析质量分（如"解析完整度 92%"）
  - **检索热度**：展示文档被检索调用的次数，帮助运营识别高价值文档
  - **文档关联图谱**：可视化展示该文档与其他文档的关联关系
  - **列表项操作菜单**：每条文档右侧 ... 菜单（快速归档、快速分享、复制链接）
- **产品影响**：文档量增长后，当前只有筛选没有排序和视图切换，信息密度和浏览效率不足

### 14.6 文档详情页（/documents/[docId]）
- **当前能力**：头部信息、三 Tabs、Review Cockpit、生命周期操作、Diagnostics
- **缺失功能**：
  - **文档批注/评论**：用户在源文件预览区直接划线批注、添加评论（类似 Google Docs）
  - **版本历史**：展示文档的历次解析/编辑版本，支持版本对比和回滚
  - **分享链接**：生成只读分享链接，设置有效期和密码（外部协作用）
  - **访问记录**："最近 10 位查看者"列表，展示谁看过、何时看过
  - **关联文档**："相关文档"推荐区（基于集合或内容相似度）
  - **导出功能**：导出为 PDF / Word / 纯文本 / Markdown
  - **打印优化**：专门的打印样式，隐藏导航和按钮，适配 A4
  - **更多预览格式**：当前仅支持 PDF/图片/HTML/文本，缺少 DOCX、PPTX 的原生预览
  - **全文搜索高亮增强**：当前仅支持单关键词搜索，应支持多关键词、正则、大小写敏感选项
- **产品影响**：文档是核心资产，缺少批注和版本管理导致无法追踪变更，缺少分享影响外部协作

### 14.7 检索验证页（/retrieval）
- **当前能力**：检索表单、证据片段展示、展开折叠、复制、调试级别选择
- **缺失功能**：
  - **检索历史**：保存最近 20 条检索记录，一键重新执行（存在 localStorage 或用户账户）
  - **结果对比**：支持同一查询用两套不同配置并行检索，左右分栏对比结果差异
  - **结果评分反馈（踩/赞）**：每条证据片段旁增加 👍/👎 按钮，收集用户反馈用于优化模型
  - **结果导出**：将检索结果导出为 JSON / Markdown / Excel（含分数和来源）
  - **检索耗时展示**：展示"检索耗时 245ms"，帮助用户感知性能
  - **空查询提示**：输入框为空时展示热门查询、最近查询、示例查询
  - **检索配置管理入口**：当前仅下拉选择，应提供"新建配置"和"编辑当前配置"的快捷入口
  - **高级检索模式**：布尔表达式（AND/OR/NOT）、字段过滤（按 doc_id 范围、按日期范围）
  - **检索结果可视化**：知识图谱展示（实体关系图），替代纯文本列表
  - **API 代码片段**：展示当前检索对应的 cURL / Python SDK 代码，方便开发者复制
  - **检索参数预设**：保存常用参数组合为"快捷模式"（如"精确模式"/"召回模式"）
- **产品影响**：检索是知识库的核心价值出口，缺少历史管理和反馈闭环无法持续优化检索质量

### 14.8 知识库集合页（/collections）
- **当前能力**：卡片网格、新建集合、选择用于上传
- **缺失功能**：
  - **集合详情页**：点击进入集合查看内部文档列表、集合概览（当前点击卡片无任何反应）
  - **集合统计**：文档总数、各状态分布饼图、存储空间占用、最近上传时间
  - **集合编辑**：修改集合名称和描述（当前创建后不可改）
  - **集合删除**：带二次确认和文档迁移选项（删除前将文档移到默认集合）
  - **集合搜索/排序**：集合多了之后需要按名称搜索，按文档数/时间排序
  - **集合权限配置**：配置谁可以上传/谁可以检索/谁可以管理（当前无权限隔离）
  - **集合访问日志**：谁在何时上传/检索了该集合
  - **集合导入/导出**：批量导入文档到集合、导出集合元数据为 JSON
  - **集合模板**：提供预设模板（"产品手册"/"法规库"/"FAQ"），一键创建带推荐配置的标准集合
  - **集合标签/分类**：给集合打分类标签（"内部"/"外部"/"合规"），支持按分类筛选
- **产品影响**：集合是知识库的组织单元，当前只建不管，无法支撑企业级多团队隔离需求

### 14.9 设置页（/settings）
- **当前能力**：认证 Tab（JWT/APIKey）、权限范围 Tab（内部/外部）
- **缺失功能**：
  - **个人资料**：头像、昵称、联系方式、所属部门
  - **偏好设置**：
    - 语言切换（中文/英文/其他）
    - 主题切换（暗色/亮色/跟随系统）
    - 界面密度（紧凑/舒适）
    - 默认侧边栏展开/收起状态
  - **安全设置**：
    - 修改密码
    - 双因素认证（2FA）
    - 登录设备管理（查看/踢出已登录设备）
    - 登录历史（IP、时间、地点）
  - **数据管理**：
    - 清除本地缓存
    - 导出个人数据（JSON 格式）
    - 导入个人设置
  - **通知偏好**：
    - 站内通知开关（按事件类型）
    - 邮件通知开关
    - 通知免打扰时段
  - **快捷键自定义**：查看和修改全局快捷键映射
  - **账户管理**：账户注销申请、数据删除请求（GDPR 合规）
  - **关于页面**：产品版本号、构建时间、开源许可证列表、服务状态链接
- **产品影响**：企业用户的账户安全、个性化偏好、合规诉求在设置页完全空白

### 14.10 全局层面
- **面包屑导航**：当前无面包屑，深层页面（如 `/documents/xxx`）用户容易迷路
- **404/500 页面**：当前无定制错误页，直接暴露框架默认错误
- **操作确认统一规范**：删除/归档/撤回等操作应统一使用 Dialog 二次确认，当前部分操作（如批量 Retract）确认弹窗较简单，缺少操作后果说明
- **表单自动保存**：审核决策原因、检索查询等输入框内容应自动保存到 localStorage，防刷新丢失
- **离线提示**：网络断开时顶部展示"网络异常，正在重试..."提示条
- **多标签页状态同步**：同一用户开多个标签页时，集合选择、认证设置应同步（BroadcastChannel）
- **键盘快捷键**：全局快捷键表（`?` 唤起），支持 `/` 聚焦页面搜索、`Esc` 关闭弹窗
- **打印样式**：所有页面的 `@media print` 优化，隐藏导航和按钮
- **无障碍（a11y）**：当前缺少足够的 `aria-label`、焦点管理、键盘导航支持
- **加载骨架屏统一**：部分页面（如检索结果）缺少加载骨架，直接空白等待
- **空状态差异化**：当前各页面 EmptyState 较统一，建议按场景差异化（如上传页空状态可引导"拖拽文件到上方区域"）
