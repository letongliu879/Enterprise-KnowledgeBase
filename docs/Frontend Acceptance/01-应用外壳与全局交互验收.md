# 01 - 应用外壳与全局交互验收

> **文档定位**：本文件验证应用级别的“容器”体验。AppShell、Header、Sidebar、全局状态、全局交互、健康检查、API 客户端等不属于某一个业务模块，但直接影响用户能否顺利进入任何业务功能。
>
> **涉及页面**：所有页面（全局生效）。
>
> **关联需求**：`需求.md` 第 1 章（全局功能）、第 14.10 节（全局层面）。

---

## 1. 验收目标与范围

### 1.1 业务目标

验证用户从打开应用到进入任意业务页面，始终能：

1. 知道自己在哪个页面、如何切换页面；
2. 感知当前所选知识库集合和系统健康状态；
3. 获得及时的操作反馈（Toast、加载、错误提示）；
4. 在出错时看到友好降级，而不是白屏或静默失败；
5. 通过全局搜索、通知中心、快捷键等效率工具快速触达功能。

### 1.2 涉及组件

| 组件/区域 | 文件路径 | 作用 |
|---|---|---|
| AppShell | `src/components/layout/app-shell.tsx` | 侧边栏 + 顶部栏 + 内容区外壳 |
| Sidebar | `src/components/layout/sidebar.tsx`（如存在） | 左侧导航 |
| Header | `src/components/layout/header.tsx`（如存在） | 顶部工具栏 |
| Providers | `src/components/providers.tsx` | Query + Tooltip Providers |
| ErrorBoundary | `src/components/error-boundary.tsx` | 全局错误边界 |
| BackendGap | `src/components/backend-gap.tsx` | 后端能力缺口提示 |
| OfflineToast | `src/components/offline-toast.tsx` | 离线提示 |
| CommandPalette | `src/components/command-palette.tsx` | 全局搜索 |
| NotificationCenter | `src/features/notifications/notification-center.tsx` | 通知中心 |
| API Client | `src/lib/api/client.ts` | 统一 API 客户端 |
| Zustand Store | `src/lib/store.ts` | 全局客户端状态 |

## 2. 用户角色、前置条件与可用性基线

### 2.1 角色

| 角色 | 关注的外壳能力 |
|---|---|
| 所有用户 | 导航、集合选择、健康状态、Toast、错误提示 |
| 管理员 | Admin lifecycle Badge、批量操作入口 |
| 移动端用户 | 抽屉式侧边栏、响应式布局 |

### 2.2 前置条件

- 应用可访问（`npm run dev` 或构建产物运行）；
- 至少一个知识库集合存在（用于验证集合选择器）；
- 后端服务至少部分可访问（用于验证健康指示灯）。

### 2.3 可用性基线

默认遵循 `00-全局可用性基线与验收规则.md`。本模块额外强调：

- 侧边栏导航必须始终可用，不能因为某个后端服务离线而消失；
- Header 健康指示灯必须真实反映后端状态；
- 全局搜索必须能通过键盘触发和导航。

## 3. 全链路场景叙事（A - 主结构）

### 场景 1：首次进入应用并导航到目标页面

**用户**：普通操作员
**目标**：从首页进入批量入库页，确认导航、集合选择、健康状态正常。
**优先级**：P0

#### 步骤 1：打开首页

**操作描述**：
1. 用户访问 `/`。

**预期界面状态**：
- 服务端重定向到 `/upload`；
- 页面加载时展示 Dashboard（若已配置）或批量入库页；
- AppShell 渲染完成：左侧侧边栏、顶部 Header、内容区、Footer。

**[B] 单点验收点**：
- `SHE-001`：访问 `/` 后 URL 变为 `/upload`；
- `SHE-002`：AppShell 三区域（侧边栏、Header、内容区）均渲染，无白屏；
- `SHE-003`：页面切换带淡入 + 上移动画。

#### 步骤 2：查看侧边栏导航

**操作描述**：
1. 用户查看左侧侧边栏；
2. 点击不同导航项切换页面。

**预期界面状态**：
- 侧边栏顶部显示 Logo + “Knowledge Workbench”；
- 导航项：批量入库、人工复核、文档库、检索验证、知识库集合；
- 当前页面高亮（背景色 + 文字色 + 右侧脉冲圆点）；
- 桌面端侧边栏可折叠/展开，带动画 width 过渡；
- 移动端汉堡菜单触发抽屉式侧边栏，遮罩层点击可关闭。

**[B] 单点验收点**：
- `SHE-004`：侧边栏所有导航项文案与需求一致；
- `SHE-005`：当前页面高亮样式正确；
- `SHE-006`：点击导航项正确跳转对应路由；
- `SHE-007`：侧边栏折叠/展开动画流畅，内容区自适应；
- `SHE-008`：移动端抽屉可打开、可关闭、可点击遮罩关闭。

**[C] 视觉规格**：
- Logo 与产品名称在侧边栏顶部居左；
- 当前页面高亮背景色与主题主色一致；
- 右侧脉冲圆点直径约 6-8px，动画柔和。

#### 步骤 3：查看顶部 Header

**操作描述**：
1. 用户查看顶部 Header；
2. 观察健康状态指示灯；
3. 切换集合选择器；
4. 点击设置入口。

**预期界面状态**：
- Header 左侧有侧边栏切换按钮（汉堡/X 图标动画切换）；
- 健康状态指示灯显示 Admin / Workbench / Access / Retrieval / Ingestion 状态；
- 全部健康时显示 “All Healthy” 徽标（绿点脉冲动画）；
- 集合选择器下拉展示可选集合，从 `/workbench/me` 获取租户；
- 设置入口按钮可点击，跳转 `/settings`。

**[B] 单点验收点**：
- `SHE-009`：健康指示灯颜色正确（健康绿色、异常红色/黄色）；
- `SHE-010`：All Healthy 徽标在全部服务健康时显示；
- `SHE-011`：集合选择器加载并展示可选集合；
- `SHE-012`：切换集合后 Zustand `currentCollectionId` 更新；
- `SHE-013`：设置入口跳转 `/settings`。

**[C] 视觉规格**：
- Header 高度固定，阴影/边框与内容区分隔；
- 健康指示灯为圆形，All Healthy 绿点带脉冲动画；
- 集合选择器触发器宽度适中，下拉内容带阴影和圆角。

### 场景 2：全局搜索与通知中心

**用户**：普通操作员
**目标**：通过全局搜索快速跳转页面，查看通知。
**优先级**：P1

#### 步骤 1：打开 Command Palette

**操作描述**：
1. 用户按 `Cmd/Ctrl + K` 或点击 Header 搜索框；
2. 输入关键词搜索页面/文档/工单/集合；
3. 使用键盘上下箭头选择，Enter 跳转。

**预期界面状态**：
- 暗色半透明遮罩覆盖页面；
- 搜索框居中显示；
- 搜索结果按类型分组（页面/文档/工单/集合）；
- 每条结果有类型标签；
- 键盘上下箭头高亮选择项，Enter 跳转，Esc/点击外部关闭。

**[B] 单点验收点**：
- `SHE-014`：`Cmd/Ctrl+K` 可唤起 Command Palette；
- `SHE-015`：搜索结果包含页面路由（7 个）、文档、工单、集合；
- `SHE-016`：键盘导航可用（上下/Enter/Esc）；
- `SHE-017`：点击外部或 Esc 关闭。

#### 步骤 2：查看通知中心

**操作描述**：
1. 用户点击 Header 右侧铃铛图标；
2. 查看通知列表；
3. 点击单条通知；
4. 点击“全部标记已读”。

**预期界面状态**：
- 铃铛图标上有未读数徽标（Badge）；
- 下拉面板以 scale + fade 动画展开；
- 通知列表展示标题、消息、时间戳；
- 已读/未读视觉区分（不透明度、蓝点指示）；
- 点击通知标记已读并跳转链接；
- “全部标记已读”按钮可清空未读徽标。

**[B] 单点验收点**：
- `SHE-018`：未读数徽标显示正确；
- `SHE-019`：通知面板动画正常；
- `SHE-020`：点击通知后标记已读并跳转；
- `SHE-021`：全部标记已读清空未读数。

### 场景 3：全局状态与错误处理

**用户**：普通操作员
**目标**：设置认证 token、权限范围，遇到错误时正确提示。
**优先级**：P0

#### 步骤 1：配置演示凭证

**操作描述**：
1. 用户进入 `/settings`；
2. 在认证 Tab 输入 JWT 令牌和 API 密钥；
3. 点击保存。

**预期界面状态**：
- 输入框为 password 类型，可切换显示/隐藏；
- 保存后更新 Zustand store；
- Toast 提示保存成功；
- 刷新页面后配置持久化（localStorage）。

**[B] 单点验收点**：
- `SHE-022`：JWT/API Key 输入框类型正确；
- `SHE-023`：保存后 Zustand 更新；
- `SHE-024`：持久化到 localStorage；
- `SHE-025`：保存后有 Toast 反馈。

#### 步骤 2：遇到后端 501 错误

**操作描述**：
1. 用户触发一个后端未实现的功能；
2. 观察页面反馈。

**预期界面状态**：
- 不白屏；
- 展示 BackendGap 组件；
- 组件明确说明“功能尚未就绪”及缺失端点；
- 提供返回或刷新入口。

**[B] 单点验收点**：
- `SHE-026`：API 返回 501 时展示 BackendGap；
- `SHE-027`：BackendGap 文案包含缺失端点信息；
- `SHE-028`：不静默失败。

#### 步骤 3：触发全局错误边界

**操作描述**：
1. 用户访问一个会触发渲染错误的页面（或模拟错误）。

**预期界面状态**：
- 不白屏；
- 展示 ErrorBoundary 友好错误页；
- 页面包含刷新按钮；
- 侧边栏和 Header 可继续保持可用（取决于实现）。

**[B] 单点验收点**：
- `SHE-029`：渲染错误被 ErrorBoundary 捕获；
- `SHE-030`：错误页文案友好，含刷新按钮。

## 4. 异常 / 容错场景

### 场景 4：后端健康检查失败

**优先级**：P1
**异常点**：部分后端服务离线。

**预期系统行为**：
- Header 健康指示灯中对应服务变红/黄；
- All Healthy 徽标消失；
- 离线服务相关功能展示 BackendGap 或禁用；
- 其他服务不受影响。

**验收点**：
- `SHE-031`：健康轮询（30s 间隔）正确更新指示灯；
- `SHE-032`：单个服务离线不影响其他服务使用。

### 场景 5：未配置 token 访问受保护功能

**优先级**：P0
**异常点**：用户未配置 JWT/API Key。

**预期系统行为**：
- 受保护操作被禁用或引导配置；
- API 401 后提示重新配置 token；
- 不能无限重试或静默失败。

**验收点**：
- `SHE-033`：未配置 token 时给出明确提示；
- `SHE-034`：401 后引导到 `/settings`。

### 场景 6：移动端侧边栏无法关闭

**优先级**：P1
**异常点**：移动端抽屉打开后无法关闭。

**预期系统行为**：
- 点击遮罩层关闭；
- 点击导航项后自动关闭；
- 点击 X 按钮关闭。

**验收点**：
- `SHE-035`：移动端抽屉至少有两种关闭方式。

### 场景 7：通知加载失败

**优先级**：P1
**异常点**：通知 API 返回错误。

**预期系统行为**：
- 面板内展示错误提示；
- 不无限 loading；
- 提供重试入口。

**验收点**：
- `SHE-036`：通知加载失败有提示；
- `SHE-037`：可重试。

### 场景 8：Command Palette 搜索无结果

**优先级**：P1
**异常点**：关键词无匹配。

**预期系统行为**：
- 提示“无匹配结果”；
- 可清空搜索重试。

**验收点**：
- `SHE-038`：无结果提示清晰；
- `SHE-039`：可清空搜索。

### 场景 9：Command Palette 移动端适配

**优先级**：P2
**异常点**：移动端屏幕尺寸小，键盘快捷键难以使用。

**预期系统行为**：
- 移动端仍可通过点击 Header 搜索框唤起；
- 弹窗适配移动端宽度。

**验收点**：
- `SHE-040`：移动端可点击唤起；
- `SHE-041`：移动端弹窗宽度适配。

## 6. 验收结论与判定规则

| 缺陷类型 | 判定规则 | 结论 |
|---|---|---|
| 功能可用缺陷 | 导航失效、token 无法保存、Health 不更新 | 失败，阻塞发布 |
| 视觉缺陷 | 高亮/动画/Badge 样式错误但不影响导航 | 记录，P0 阻塞 |
| 流程链路缺陷 | 全局搜索无法跳转、通知点击不跳转 | 失败，阻塞发布 |
| 数据一致性缺陷 | Header 集合选择器与 /collections /upload 状态不同步、Health 状态与各页面不一致 | 失败，阻塞发布 |
| 已知缺口 | 面包屑导航未实现 | 失败（Not Implemented） |
