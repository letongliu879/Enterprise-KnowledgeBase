# 10 - 首页与仪表盘验收

> **文档定位**：本文件验证首页（`/`）和仪表盘（Dashboard）功能的完整用户体验。首页服务端重定向到 `/upload`，但仪表盘作为运营/导航入口，提供全局状态概览和快捷操作。
>
> **涉及页面**：`/`（重定向到 `/upload`）、`/upload` 中的 Dashboard 组件。
>
> **关联需求**：`需求.md` 第 2 章、第 13.1 节。

---

## 1. 验收目标与范围

### 1.1 业务目标

验证用户能够：

1. 访问 `/` 时正确重定向到 `/upload`；
2. 在 Dashboard 中看到当前工作台的概览数据；
3. 通过快捷操作卡片快速进入各功能模块；
4. 查看最近工单并快速跳转复核；
5. 看到公告/运营 Banner；
6. 在加载、错误、空状态时获得友好反馈。

### 1.2 涉及页面

| 路由 | 页面 | 作用 |
|---|---|---|
| `/` | 首页 | 服务端重定向到 `/upload` |
| `/upload` | 批量入库页 | 包含 Dashboard 组件 |

### 1.3 涉及角色

- **所有用户**：查看 Dashboard、使用快捷操作。

---

## 2. 用户角色、前置条件与可用性基线

### 2.1 角色

| 角色 | 使用场景 |
|---|---|
| 操作员 | 查看上传量、待复核数、快捷操作 |
| 复核员 | 查看待复核数、最近工单 |
| 管理员 | 查看文档总数、Stale 比例 |

### 2.2 前置条件

- 应用可访问；
- 后端部分服务可访问（用于加载 Dashboard 数据）。

### 2.3 可用性基线

默认遵循 `00-全局可用性基线与验收规则.md`。本模块额外强调：

- Dashboard 数据加载必须有骨架屏；
- 快捷操作卡片必须可点击并正确跳转；
- 最近工单列表必须能点击进入复核详情。

## 3. 全链路场景叙事（A - 主结构）

### 场景 1：用户进入首页并被重定向

**用户**：普通用户
**目标**：访问应用根路径后进入批量入库页。
**优先级**：P0

#### 步骤 1：访问根路径

**操作描述**：
1. 用户在浏览器地址栏输入 `/` 或点击 Logo 返回首页。

**预期界面状态**：
- 服务端返回 307/302 重定向到 `/upload`；
- 浏览器 URL 变为 `/upload`；
- `/upload` 页面渲染，包含 Dashboard 组件。

**[B] 单点验收点**：
- `DASH-001`：访问 `/` 后 URL 变为 `/upload`；
- `DASH-002`：不重定向循环、不白屏；
- `DASH-003`：未认证用户也能被正确引导（如跳登录或展示设置引导）。

### 场景 2：用户查看仪表盘概览

**用户**：操作员
**目标**：通过 Dashboard 快速了解当前工作台状态。
**优先级**：P0

#### 步骤 1：查看问候语

**操作描述**：
1. 用户进入 `/upload`，顶部展示时区感知问候语。

**预期界面状态**：
- 根据当前时间显示“早上好/下午好/晚上好”；
- 问候语包含用户名称或通用称呼。

**[B] 单点验收点**：
- `DASH-004`：问候语按时区正确显示；
- `DASH-005`：不同时段文案正确。

#### 步骤 2：查看统计卡片

**操作描述**：
1. 用户查看 4 个统计卡片。

**预期界面状态**：
- 4 个统计卡片展示：今日上传量、待复核数、文档总数、Stale 比例；
- 数字加载完成后显示；
- 卡片有图标、标签、数字。

**[B] 单点验收点**：
- `DASH-006`：4 个统计卡片展示；
- `DASH-007`：数字与后端数据一致；
- `DASH-008`：加载时展示 Skeleton。

**[C] 视觉规格**：
- 4 张卡片等宽或响应式排列；
- 数字使用大号字体；
- Stale 比例卡片使用 warning/destructive 颜色突出。

#### 步骤 3：使用快捷操作

**操作描述**：
1. 用户点击 6 个快捷操作卡片之一。

**预期界面状态**：
- 6 个快捷操作卡片：上传、复核、文档库、检索、集合、设置；
- 点击后跳转对应页面。

**[B] 单点验收点**：
- `DASH-009`：6 个快捷操作卡片展示；
- `DASH-010`：点击后正确跳转对应路由。

**[C] 视觉规格**：
- 卡片采用图标 + 标题 + 描述布局；
- hover 时有阴影/位移反馈；
- 卡片颜色或图标区分功能。

#### 步骤 4：查看最近工单

**操作描述**：
1. 用户查看最近工单列表；
2. 点击某条工单跳转复核详情。

**预期界面状态**：
- 列表展示最近若干条工单（如 5-10 条）；
- 每条工单显示文件名、状态、时间；
- 点击跳转 `/review/[taskId]`。

**[B] 单点验收点**：
- `DASH-011`：最近工单列表展示；
- `DASH-012`：点击跳转复核详情；
- `DASH-013`：无工单时展示空状态或隐藏该区块。

#### 步骤 5：查看公告/运营 Banner

**操作描述**：
1. 用户查看页面上方或侧边的 Banner。

**预期界面状态**：
- Banner 展示公告或运营内容；
- 可关闭（如实现）。

**[B] 单点验收点**：
- `DASH-014`：Banner 展示；
- `DASH-015`：内容可读取。

## 4. 异常 / 容错场景

### 场景 3：Dashboard 数据加载失败

**优先级**：P1
**异常点**：后端返回错误或超时。

**预期系统行为**：
- 展示错误 Alert；
- 不白屏；
- 提供刷新按钮。

**验收点**：
- `DASH-ERR-001`：加载失败展示 Alert；
- `DASH-ERR-002`：提供刷新入口。

### 场景 4：无数据时的 Dashboard

**优先级**：P1
**异常点**：新系统没有任何上传/工单/文档。

**预期系统行为**：
- 统计卡片显示 0；
- 最近工单区块展示 EmptyState 或隐藏；
- 快捷操作卡片仍然可用。

**验收点**：
- `DASH-ERR-003`：零数据时统计为 0；
- `DASH-ERR-004`：最近工单空状态友好。

### 场景 5：重定向异常

**优先级**：P0
**异常点**：访问 `/` 后未正确跳转。

**预期系统行为**：
- 必须重定向到 `/upload`；
- 不能出现 404 或循环重定向。

**验收点**：
- `DASH-ERR-005`：根路径重定向正确。

## 6. 验收结论与判定规则

| 缺陷类型 | 判定规则 | 结论 |
|---|---|---|
| 功能可用缺陷 | 首页不跳转、统计卡片不加载、快捷操作不跳转 | 失败，阻塞发布 |
| 视觉缺陷 | 卡片布局/数字样式异常 | 记录，P0 阻塞 |
| 流程链路缺陷 | 最近工单点击后 ID 错误、数据与后端不一致 | 失败，阻塞发布 |
| 数据一致性缺陷 | Dashboard 统计数字与各页面明细汇总不一致 | 失败，阻塞发布 |
