# 08 - 知识库集合验收

> **文档定位**：本文件验证知识库集合页（`/collections`）的集合列表、创建/编辑、选择用于上传、集合统计、搜索等能力。集合是上传、检索等业务的前置上下文。
>
> **涉及页面**：`/collections`、`/collections/[collectionId]`（详情页）。
>
> **关联需求**：`需求.md` 第 9 章、第 14.8 节。

---

## 1. 验收目标与范围

### 1.1 业务目标

验证用户能够：

1. 查看所有可访问的知识库集合；
2. 创建新的集合；
3. 编辑集合名称和描述；
4. 选择某个集合作为当前操作集合；
5. 查看集合的基本统计信息；
6. 在空状态时创建第一个集合。

### 1.2 涉及页面

| 路由 | 页面 | 作用 |
|---|---|---|
| `/collections` | 知识库集合页 | 集合列表、创建、选择 |
| `/collections/[collectionId]` | 集合详情页 | 集合统计、编辑、搜索 |

### 1.3 涉及角色

- **操作员**：查看集合、选择集合用于上传；
- **管理员**：创建/编辑集合。

---

## 2. 用户角色、前置条件与可用性基线

### 2.1 角色与权限

| 角色 | 权限 | 本页面使用场景 |
|---|---|---|
| 操作员 | 查看集合、选择集合 | 主要用户 |
| 管理员 | 创建/编辑集合 | 集合管理 |

### 2.2 前置条件

- 已配置有效 token；
- 后端 `/workbench/me` 可返回租户信息；
- 测试环境可创建集合（管理员权限）。

### 2.3 可用性基线

默认遵循 `00-全局可用性基线与验收规则.md`。本模块额外强调：

- 集合选择必须即时同步到 Header 和 Zustand；
- 创建集合必须依赖租户 ID，未获取到租户时给出明确提示；
- 集合详情页的统计信息必须准确反映后端数据。

## 3. 全链路场景叙事（A - 主结构）

### 场景 1：操作员选择集合用于上传

**用户**：操作员
**目标**：在集合列表中选择一个集合，作为后续上传和检索的上下文。
**优先级**：P0

#### 步骤 1：进入集合页

**操作描述**：
1. 操作员点击侧边栏“知识库集合”进入 `/collections`。

**预期界面状态**：
- 页面加载时展示 3 个骨架屏卡片；
- 数据加载完成后展示集合卡片网格（1-3 列响应式）；
- 每张卡片显示：
  - 文件夹图标 + 集合名称；
  - 生命周期状态 Badge（active/其他）；
  - 集合 ID（monospace，小号字）；
  - 描述（2 行截断，无描述显示 “无描述”）；
  - 租户 ID；
  - “选择用于上传”按钮。

**[B] 单点验收点**：
- `COL-001`：加载时展示 3 个骨架屏卡片；
- `COL-002`：卡片网格为 1-3 列响应式；
- `COL-003`：卡片展示名称、状态 Badge、ID、描述、租户 ID；
- `COL-004`：无描述时显示“无描述”。

**[C] 视觉规格**：
- 卡片网格间距一致；
- 文件夹图标与集合名称左对齐；
- 集合 ID 使用小号等宽字体；
- 描述 2 行截断，溢出显示省略号。

#### 步骤 2：选择集合

**操作描述**：
1. 操作员点击目标卡片上的“选择用于上传”按钮。

**预期界面状态**：
- 按钮点击后有加载/成功反馈；
- Header 集合选择器同步更新为当前集合；
- Toast 提示“已选择集合：XXX”；
- Zustand `currentCollectionId` 更新。

**[B] 单点验收点**：
- `COL-005`：点击后 Toast 提示成功；
- `COL-006`：Header 选择器同步更新；
- `COL-007`：Zustand currentCollectionId 更新；
- `COL-008`：切换到 `/upload` 后上传区不再因未选集合被禁用。

**[C] 视觉规格**：
- 选中集合卡片有视觉反馈（如边框高亮）；
- Toast 位置符合全局 Toast 规范。

### 场景 2：管理员创建新集合

**用户**：管理员
**目标**：创建一个新的知识库集合。
**优先级**：P0

#### 步骤 1：打开创建弹窗

**操作描述**：
1. 管理员点击“新建集合”按钮；
2. 弹窗打开。

**预期界面状态**：
- 弹窗为 glass 风格圆角 Dialog；
- 包含字段：集合 ID、名称、描述；
- 提交按钮默认 disabled，必填项填写后启用。

**[B] 单点验收点**：
- `COL-009`：新建按钮可点击；
- `COL-010`：弹窗字段与需求一致；
- `COL-011`：必填项校验生效。

#### 步骤 2：填写并提交

**操作描述**：
1. 管理员填写集合 ID、名称、描述；
2. 点击提交；
3. 观察结果。

**预期界面状态**：
- 提交按钮依赖租户 ID（从 `/workbench/me` 获取）；
- 提交成功后刷新列表、关闭弹窗、重置表单；
- Toast 提示创建成功；
- 后端缺口时展示 BackendGap。

**[B] 单点验收点**：
- `COL-012`：提交依赖租户 ID；
- `COL-013`：创建成功后列表刷新；
- `COL-014`：弹窗关闭、表单重置；
- `COL-015`：Toast 提示成功；
- `COL-016`：后端缺口时展示 BackendGap。

### 场景 3：编辑集合信息

**用户**：管理员
**目标**：修改已有集合的名称和描述。
**优先级**：P1

**预期界面状态**：
- 集合卡片或详情页有编辑入口；
- 编辑弹窗预填充当前名称和描述；
- 保存后列表/详情同步更新。

**[B] 单点验收点**：
- `COL-017`：编辑入口可用；
- `COL-018`：弹窗预填充数据；
- `COL-019`：保存后列表更新。

### 场景 4：查看集合详情

**用户**：操作员/管理员
**目标**：查看某个集合的统计信息和文档列表。
**优先级**：P1

**预期界面状态**：
- 点击集合卡片进入 `/collections/[collectionId]`；
- 详情页展示集合统计：文档数、Chunks/Pages、最后上传、平均 Chunk 数；
- 可搜索集合内文档；
- 详情页有编辑/权限/导入/导出/模板/标签按钮（部分可能占位）。

**[B] 单点验收点**：
- `COL-020`：详情页统计展示正确；
- `COL-021`：集合内文档可搜索；
- `COL-022`：统计数字与后端一致。

## 4. 异常 / 容错场景

### 场景 5：集合列表为空

**优先级**：P0
**异常点**：当前租户下没有任何集合。

**预期系统行为**：
- 展示 EmptyState + 创建按钮；
- 提示用户创建第一个集合。

**验收点**：
- `COL-ERR-001`：空状态展示 EmptyState；
- `COL-ERR-002`：提供创建按钮。

### 场景 6：创建集合失败

**优先级**：P0
**异常点**：集合 ID 已存在或后端返回错误。

**预期系统行为**：
- 表单字段级错误提示（如 ID 重复）；
- 或 Toast 提示通用错误；
- 弹窗不关闭，保留用户输入。

**验收点**：
- `COL-ERR-003`：ID 重复有明确提示；
- `COL-ERR-004`：失败时不丢失用户输入。

### 场景 7：无法获取租户 ID

**优先级**：P0
**异常点**：`/workbench/me` 失败。

**预期系统行为**：
- 新建集合按钮 disabled 或点击后提示；
- 展示 BackendGap 或错误提示。

**验收点**：
- `COL-ERR-005`：无租户信息时阻止创建；
- `COL-ERR-006`：用户能感知原因。

### 场景 8：集合搜索无结果

**优先级**：P1
**异常点**：按名称搜索集合无匹配。

**预期系统行为**：
- 提示“无匹配集合”；
- 提供清除搜索入口。

**验收点**：
- `COL-ERR-007`：无匹配提示清晰。

## 6. 验收结论与判定规则

| 缺陷类型 | 判定规则 | 结论 |
|---|---|---|
| 功能可用缺陷 | 无法选择集合、无法创建集合、Header 不同步 | 失败，阻塞发布 |
| 视觉缺陷 | 卡片网格/描述截断/Badge 样式异常 | 记录，P0 阻塞 |
| 流程链路缺陷 | 选择集合后上传仍提示未选、创建后列表不刷新 | 失败，阻塞发布 |
| 数据一致性缺陷 | /collections 选中状态与 Header 选择器、/upload 前置校验不一致 | 失败，阻塞发布 |
