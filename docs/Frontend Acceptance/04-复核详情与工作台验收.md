# 04 - 复核详情与工作台验收

> **文档定位**：本文件验证复核详情页（`/review/[taskId]`）和工作台详情页（`/workbench/[ticketId]`）的完整用户体验。这是人工复核的核心工作界面，涉及源文件预览、Chunk 草稿编辑、Agent 审核发现、审批决策、工单上下文及诊断信息。
>
> **涉及页面**：`/review/[taskId]`、`/workbench/[ticketId]`。
>
> **关联需求**：`需求.md` 第 5 章、第 13.6 节、第 14.4 节。

---

## 1. 验收目标与范围

### 1.1 业务目标

验证复核员能够：

1. 查看工单的完整上下文（状态、指标、元数据）；
2. 在 Source Tab 预览原始文件和解析后文本；
3. 在 Draft edits Tab 查看并编辑 chunks；
4. 在 Agent review Tab 查看 AI 发现的审核问题；
5. 根据审查结果做出 Approve/Reject/Return 决策；
6. 查看工单评论、时间轴、诊断信息；
7. 当 workspace 部分数据不可用时，能看到降级提示。

### 1.2 涉及页面

| 路由 | 页面 | 作用 |
|---|---|---|
| `/review/[taskId]` | 复核详情页 | 人工审批决策 |
| `/workbench/[ticketId]` | 工作台详情页 | 与复核详情共享组件，Workspace 聚合视图 |

### 1.3 涉及角色

- **复核员**：查看工单、编辑 chunks、审批决策；
- **操作员**：查看自己上传文件生成的工单；
- **管理员**：查看系统决策、失败码、诊断信息。

---

## 2. 用户角色、前置条件与可用性基线

### 2.1 角色与权限

| 角色 | 权限 | 本页面使用场景 |
|---|---|---|
| 复核员 | 查看详情、编辑 chunks、Approve/Reject/Return | 主要用户 |
| 操作员 | 查看详情（可能无审批权限） | 追踪结果 |
| 管理员 | 查看诊断、系统决策 | 问题排查 |

### 2.2 前置条件

- 已配置有效 token；
- 队列中存在至少一条待复核或已完成复核的工单；
- 测试工单最好包含：源文件、解析快照、chunks、agent_review findings（用于验证三个 Tab）。

### 2.3 可用性基线

默认遵循 `00-全局可用性基线与验收规则.md`。本模块额外强调：

- 三个 Tab 必须可稳定切换，切换时不能丢失已加载内容；
- PDF 预览、Chunk 编辑、Finding 定位等重交互区域必须有明确的加载和错误反馈；
- 审批决策是破坏性操作，必须有二次确认。

## 3. 全链路场景叙事（A - 主结构）

### 场景 1：复核员完整审批一条工单

**用户**：复核员
**目标**：查看工单详情、审查源文件和 Agent 发现、编辑 chunk、做出审批决策。
**优先级**：P0

#### 步骤 1：进入复核详情页

**操作描述**：
1. 复核员从 `/review` 点击工单卡片；
2. 进入 `/review/[taskId]`。

**预期界面状态**：
- 页面加载时展示骨架屏；
- 数据加载完成后展示 Hero Section、三个 Tab、右侧边栏；
- Hero Section 包含：
  - 返回按钮（链接到复核队列）；
  - 状态徽标组（工单状态、失败码、下一步动作、文档关联来源）；
  - 文档标题（回退链：文件名 → doc_id → ticket_id）；
  - 描述文案；
  - 四宫格指标卡片（Findings 数、Risks、Draft edits 数、Warnings 数）；
  - 元数据网格（工单 ID、集合 ID、文档 ID、解析快照 ID）。

**[B] 单点验收点**：
- `RVD-001`：页面通过 `workbenchApi.getWorkspaceDetail(ticketId)` 聚合加载数据；
- `RVD-002`：Hero Section 四宫格指标与元数据网格展示正确；
- `RVD-003`：返回按钮跳转 `/review`；
- `RVD-004`：文档标题回退链正确。

**[C] 视觉规格**：
- Hero Section 四宫格卡片等宽，数字大号显示；
- 元数据网格 label 与 value 对齐；
- 状态徽标组间距一致，Badge 颜色正确。

#### 步骤 2：查看 Source Tab 源文件

**操作描述**：
1. 复核员点击“Source” Tab；
2. 使用工具栏缩放、翻页、复制解析文本、下载源文件；
3. 切换到“Parsed text”子 Tab。

**预期界面状态**：
- Source Tab 默认展示原始文件；
- PDF 通过 Canvas 渲染，支持分页 + 缩放（50%-200%）；
- 图片直接预览；
- HTML 通过 iframe 渲染；
- 纯文本直接展示；
- 工具栏有缩放控件、复制解析文本按钮、下载源文件按钮；
- 分页导航有上一页/下一页、页码输入跳转；
- 锚定页按钮根据 chunks 的 page_spans 生成快捷跳转；
- Parsed text 子 Tab 展示解析后的文本内容；
- 无源文件时展示 Alert 提示。

**[B] 单点验收点**：
- `RVD-005`：Source Tab 支持 PDF/图片/HTML/文本；
- `RVD-006`：PDF 缩放范围 50%-200%；
- `RVD-007`：分页导航可用，页码输入跳转生效；
- `RVD-008`：锚定页按钮根据 page_spans 生成；
- `RVD-009`：Parsed text 子 Tab 可切换；
- `RVD-010`：无源文件时展示 Alert。

**[C] 视觉规格**：
- 文档查看器占据主内容区大部分宽度；
- 工具栏固定在查看器顶部或底部；
- PDF Canvas 清晰，缩放时无模糊（在合理范围内）。

#### 步骤 3：查看 Draft edits Tab 并编辑 Chunk

**操作描述**：
1. 复核员点击“Draft edits” Tab；
2. 在搜索框输入内容或 evidence_id 搜索 chunks；
3. 展开/折叠长内容；
4. 点击编辑按钮，打开 ChunkEditModal；
5. 修改内容后提交到草稿层。

**预期界面状态**：
- Draft edits Tab 仅在有解析快照时可用；
- ChunkEditorWorkbench 组件展示 chunk 列表；
- 每条 chunk 显示 evidence_id Badge、section_path、页码范围；
- 内容 > 300 字符时可展开/折叠；
- 编辑按钮打开 ChunkEditModal；
- Modal 区分预发布模式（pre-publish：橙色主题，按钮“Draft + Submit”）；
- 编辑提交后不直接修改检索索引，而是提交到草稿层。

**[B] 单点验收点**：
- `RVD-011`：无解析快照时 Draft edits Tab 禁用或提示；
- `RVD-012`：可按内容/evidence_id 搜索 chunks；
- `RVD-013`：长内容可展开折叠；
- `RVD-014`：ChunkEditModal 可打开并编辑；
- `RVD-015`：pre-publish 模式提交到草稿层，不直接改索引；
- `RVD-016`：从 Agent Review 点击“Edit chunk”自动滚动定位到对应 chunk。

**[C] 视觉规格**：
- evidence_id Badge 使用 secondary 或 outline 样式；
- 展开/折叠按钮位置固定；
- ChunkEditModal 为 glass 风格圆角弹窗，pre-publish 用橙色强调。

#### 步骤 4：查看 Agent review Tab

**操作描述**：
1. 复核员点击“Agent review” Tab；
2. 查看 findings 列表；
3. 展开某条 finding；
4. 点击“Find in document”或“Edit chunk”。

**预期界面状态**：
- findings 按严重程度排序：Critical → High → Medium → Low → Info；
- 每条 Finding 卡片显示：严重程度 Badge、分类 Badge、置信度、问题摘要；
- 展开后展示 Source quote（引用块）、“Find in document”按钮、“Edit chunk”按钮、页码范围；
- 点击“Find in document”自动在 Source Tab 搜索并高亮定位；
- 点击“Edit chunk”自动滚动到 Draft edits Tab 对应 chunk。

**[B] 单点验收点**：
- `RVD-017`：findings 按严重程度排序；
- `RVD-018`：Finding 卡片展示 severity/category/confidence/summary；
- `RVD-019`：展开后展示 Source quote 和操作按钮；
- `RVD-020`：“Find in document”自动搜索高亮定位；
- `RVD-021`：“Edit chunk”自动滚动定位。

**[C] 视觉规格**：
- Severity Badge 颜色：Critical 红色、High 橙色、Medium 黄色、Low 蓝色、Info 灰色；
- Source quote 使用引用块样式（左侧竖线或背景色）；
- 操作按钮分组清晰。

#### 步骤 5：使用右侧 Decision Cockpit 审批

**操作描述**：
1. 复核员查看右侧 Decision Cockpit；
2. 若工单处于待复核状态且有权限，输入决策原因（可选）；
3. 点击 Approve / Reject / Return for revision；
4. 确认操作。

**预期界面状态**：
- Decision Cockpit 展示当前状态；
- 待复核 + 有权限时显示三个操作按钮和原因输入框；
- 点击按钮后弹出确认 Dialog；
- 决策成功后状态更新，按钮消失或变为“已决策”；
- 决策原因展示在 Decision Cockpit 中（或 “No additional reason”）；
- 右侧同时展示 Ticket Context 和 System & Diagnostics。

**[B] 单点验收点**：
- `RVD-022`：Approve/Reject/Return 按钮在正确状态下显示；
- `RVD-023`：无权限用户看不到决策按钮；
- `RVD-024`：决策有二次确认；
- `RVD-025`：决策原因可输入并展示；
- `RVD-026`：决策成功后状态同步更新。

**[C] 视觉规格**：
- Decision Cockpit 在右侧边栏顶部，视觉权重最高；
- Approve 按钮使用 primary/success，Reject/Return 使用 destructive；
- 原因输入框高度适中，有占位文案。

## 4. 异常 / 容错场景

### 场景 3：Workspace 数据降级

**优先级**：P1
**异常点**：`getWorkspaceDetail` 返回了 `degraded_parts`。

**预期系统行为**：
- 页面主内容区顶部展示 Alert；
- Alert 列出 degraded_parts；
- 已加载的部分正常展示；
- 缺失的部分给出明确提示。

**验收点**：
- `RVD-ERR-001`：degraded_parts 存在时展示 Alert；
- `RVD-ERR-002`：Alert 内容包含具体降级部件；
- `RVD-ERR-003`：页面不白屏。

### 场景 4：工单无源文件

**优先级**：P1
**异常点**：该工单关联的文档没有源文件。

**预期系统行为**：
- Source Tab 展示 Alert 提示“无源文件”；
- 其他 Tab（Draft edits / Agent review）仍可正常展示；
- 审批功能不受影响（如果业务允许）。

**验收点**：
- `RVD-ERR-004`：无源文件时 Source Tab 有 Alert；
- `RVD-ERR-005`：其他 Tab 不受影响。

### 场景 5：无权限审批

**优先级**：P0
**异常点**：用户没有审批权限但进入详情页。

**预期系统行为**：
- Decision Cockpit 不展示 Approve/Reject/Return 按钮；
- 可能显示“您无权限审批此工单”提示；
- 查看功能不受影响。

**验收点**：
- `RVD-ERR-006`：无权限时隐藏决策按钮；
- `RVD-ERR-007`：用户不会误以为自己可以审批。

### 场景 6：后端返回 501

**优先级**：P0
**异常点**：某个依赖服务返回 501。

**预期系统行为**：
- 对应功能区域展示 BackendGap；
- 不影响其他可加载区域；
- 用户能明确知道哪个功能未就绪。

**验收点**：
- `RVD-ERR-008`：501 时展示 BackendGap；
- `RVD-ERR-009`：文案说明缺失端点。
