# 09b - 系统管理验收

> **文档定位**：本文件验证设置页下的系统管理子页面，包括 API 密钥管理、审计日志、解析策略管理，以及偏好设置、安全设置、数据管理、通知偏好、账户管理、关于页面等。这些功能面向管理员或高级用户，与核心认证/权限解耦。
>
> **涉及页面**：`/settings/api-keys`、`/settings/audit-log`、`/settings/parser-profiles`、以及 `/settings` 其他 Tabs。
>
> **关联需求**：`需求.md` 第 13.3 节、第 13.4 节、第 13.7 节、第 14.9 节。

---

## 1. 验收目标与范围

### 1.1 业务目标

验证用户能够：

1. 管理 API 密钥（创建、编辑、吊销、查看用量统计）；
2. 查看审计日志并导出；
3. 管理解析策略（parser profiles）的 CRUD、克隆、发布；
4. 配置个人偏好（主题、语言、密度、侧边栏）；
5. 管理安全设置（密码、2FA、设备、登录历史）；
6. 管理数据（清除缓存、导出、导入）；
7. 配置通知偏好（站内/邮件/DND）；
8. 查看账户管理和关于页面。

### 1.2 涉及页面

| 路由 | 页面 | 作用 |
|---|---|---|
| `/settings/api-keys` | API 密钥管理 | 密钥 CRUD、用量统计 |
| `/settings/audit-log` | 审计日志 | 操作日志筛选、导出 |
| `/settings/parser-profiles` | 解析策略管理 | parser profile CRUD |
| `/settings` 其他 Tabs | 偏好/安全/数据/通知/账户/关于 | 用户级设置 |

### 1.3 涉及角色

- **管理员**：管理 API 密钥、解析策略；
- **操作员**：查看审计日志、配置个人设置；
- **普通用户**：配置偏好、安全、通知等。

---

## 2. 用户角色、前置条件与可用性基线

### 2.1 角色与权限

| 角色 | 权限 | 本页面使用场景 |
|---|---|---|
| 管理员 | API 密钥 CRUD、解析策略 CRUD | 系统管理 |
| 操作员 | 查看审计日志、配置个人设置 | 日常管理 |
| 普通用户 | 个人偏好/安全/数据/通知/账户 | 个人设置 |

### 2.2 前置条件

- 已配置有效 token；
- 管理员账号用于验证 API 密钥/解析策略管理。

### 2.3 可用性基线

默认遵循 `00-全局可用性基线与验收规则.md`。本模块额外强调：

- 吊销 API 密钥、彻底删除等破坏性操作必须有二次确认；
- 审计日志表格在大数据量下不能卡顿；
- 解析策略发布/克隆必须明确反馈。

## 3. 全链路场景叙事（A - 主结构）

### 场景 1：管理员管理 API 密钥

**用户**：管理员
**目标**：创建、编辑、吊销 API 密钥，并查看用量统计。
**优先级**：P1

#### 步骤 1：进入 API 密钥页

**操作描述**：
1. 管理员进入 `/settings/api-keys`。

**预期界面状态**：
- 页面展示密钥列表：名称、前缀、状态 Badge、权限标签、过期时间、最后使用时间；
- 有创建按钮。

**[B] 单点验收点**：
- `ADM-001`：密钥列表字段正确；
- `ADM-002`：状态 Badge 颜色正确。

#### 步骤 2：创建密钥

**操作描述**：
1. 管理员点击创建按钮；
2. 填写名称、选择权限；
3. 创建后复制完整密钥。

**预期界面状态**：
- 创建弹窗有权限复选框；
- 创建后展示完整密钥 + 一键复制；
- 复制后有成功反馈。

**[B] 单点验收点**：
- `ADM-003`：创建弹窗有权限选择；
- `ADM-004`：创建后展示完整密钥；
- `ADM-005`：一键复制可用。

#### 步骤 3：编辑与吊销

**操作描述**：
1. 管理员编辑密钥名称或权限；
2. 吊销某个密钥。

**预期界面状态**：
- 编辑后列表更新；
- 吊销有确认弹窗；
- 吊销后状态变为 Revoked。

**[B] 单点验收点**：
- `ADM-006`：编辑后列表更新；
- `ADM-007`：吊销需确认；
- `ADM-008`：吊销后状态更新。

#### 步骤 4：查看用量统计

**操作描述**：
1. 管理员点击某个密钥的用量统计入口。

**预期界面状态**：
- 弹窗展示总请求数、总 Token、峰值 QPS、每日统计表。

**[B] 单点验收点**：
- `ADM-009`：用量统计弹窗可打开；
- `ADM-010`：统计数据展示完整。

### 场景 2：管理员管理解析策略

**用户**：管理员
**目标**：创建、编辑、发布、克隆解析策略。
**优先级**：P1

**预期界面状态**：
- `/settings/parser-profiles` 展示策略列表；
- 列表展示名称、描述、发布/草稿 Badge；
- 支持完整 CRUD + 克隆 + 发布；
- 配置字段：parser_id、OCR、表格检测、语言。

**[B] 单点验收点**：
- `ADM-011`：策略列表字段正确；
- `ADM-012`：CRUD 可用；
- `ADM-013`：克隆/发布可用；
- `ADM-014`：配置字段正确。

### 场景 3：查看审计日志

**用户**：操作员/管理员
**目标**：查看系统操作日志并导出。
**优先级**：P1

**预期界面状态**：
- `/settings/audit-log` 展示筛选器和分页表格；
- 筛选器：操作类型（9 种）、日期范围、文档 ID、集合 ID、操作人 ID；
- 表格列：日志 ID、操作人邮箱、操作类型（彩色 Badge）、目标类型/ID、时间戳、IP；
- 页大小选择（10/20/50/100）、分页导航；
- CSV / Excel 导出按钮。

**[B] 单点验收点**：
- `ADM-015`：筛选器 9 种操作类型可用；
- `ADM-016`：分页和页大小选择可用；
- `ADM-017`：CSV/Excel 导出可用。

### 场景 4：用户配置个人偏好

**用户**：普通用户
**目标**：调整主题、语言、密度、侧边栏等偏好。
**优先级**：P2

**预期界面状态**：
- 偏好设置 Tab 可切换；
- 支持暗色/亮色/跟随系统、中文/英文、紧凑/舒适；
- 保存后立即生效或刷新后生效。

**[B] 单点验收点**：
- `ADM-018`：主题切换可用；
- `ADM-019`：语言切换可用；
- `ADM-020`：密度/侧边栏设置可用。

### 场景 5：用户管理安全与账户

**用户**：普通用户
**目标**：修改密码、查看 2FA/设备/登录历史、注销账户。
**优先级**：P2

**预期界面状态**：
- 安全设置 Tab 展示密码修改、2FA、设备管理、登录历史；
- 密码强度校验生效；
- 账户管理提供注销/数据删除入口，需二次确认。

**[B] 单点验收点**：
- `ADM-021`：密码强度校验生效；
- `ADM-022`：注销/数据删除需二次确认。

## 4. 异常 / 容错场景

### 场景 6：API 密钥创建失败

**优先级**：P1
**异常点**：后端返回错误导致密钥创建失败。

**预期系统行为**：
- Toast 提示错误原因；
- 弹窗不关闭，保留用户输入；
- 不生成虚假密钥。

**验收点**：
- `ADM-ERR-001`：创建失败有错误提示；
- `ADM-ERR-002`：不关闭弹窗丢失输入。

### 场景 7：审计日志导出失败

**优先级**：P1
**异常点**：导出 CSV/Excel 时后端错误。

**预期系统行为**：
- 提示导出失败；
- 不下载损坏文件。

**验收点**：
- `ADM-ERR-003`：导出失败有提示。

### 场景 8：解析策略发布冲突

**优先级**：P1
**异常点**：发布一个已有草稿版本的策略时冲突。

**预期系统行为**：
- 提示冲突或覆盖确认；
- 状态正确更新为已发布。

**验收点**：
- `ADM-ERR-004`：发布冲突有提示；
- `ADM-ERR-005`：状态正确更新。

### 场景 9：非管理员访问管理功能

**优先级**：P0
**异常点**：普通用户访问 API 密钥/解析策略管理。

**预期系统行为**：
- 页面提示无权限；
- 或菜单入口不展示。

**验收点**：
- `ADM-ERR-006`：非管理员无法访问管理功能。

## 6. 验收结论与判定规则

| 缺陷类型 | 判定规则 | 结论 |
|---|---|---|
| 功能可用缺陷 | API 密钥管理失败、审计日志不展示、解析策略 CRUD 失败 | 失败，阻塞发布 |
| 视觉缺陷 | Badge/表格/弹窗样式异常 | 记录，P0 阻塞 |
| 数据一致性缺陷 | 操作后日志未及时记录、状态未同步 | 失败，阻塞发布 |
| 流程链路缺陷 | 创建密钥后用量统计不关联 | 失败，阻塞发布 |
