# 06 - 文档详情验收

> **文档定位**：本文件验证文档详情页（`/documents/[docId]`）的完整用户体验。与复核详情页共享组件，但面向已发布文档的生命周期管理，强调 Review Cockpit、Lifecycle Actions、Diagnostics 和发布后的 Chunk 编辑。
>
> **涉及页面**：`/documents/[docId]`。
>
> **关联需求**：`需求.md` 第 7 章、第 14.6 节。

---

## 1. 验收目标与范围

### 1.1 业务目标

验证用户能够：

1. 查看已发布文档的完整状态、指标和元数据；
2. 在 Source / Chunks / Agent 三个 Tab 中查看内容与审核发现；
3. 对已发布文档直接编辑 chunks（post-publish 模式）；
4. 执行生命周期操作：归档、撤回、重建索引；
5. 分享文档、导出 JSON、查看相关文档推荐；
6. 在右侧 Review Cockpit 中查看关联复核工单的状态和决策。

### 1.2 涉及页面

| 路由 | 页面 | 作用 |
|---|---|---|
| `/documents/[docId]` | 文档详情页 | 已发布文档生命周期管理 |

### 1.3 涉及角色

- **操作员**：查看文档、分享、导出；
- **管理员**：执行 Archive / Retract / Reindex；
- **复核员**：查看关联复核工单状态。

---

## 2. 用户角色、前置条件与可用性基线

### 2.1 角色与权限

| 角色 | 权限 | 本页面使用场景 |
|---|---|---|
| 操作员 | 查看文档、分享、导出 JSON | 主要用户 |
| 管理员 | can_archive / can_retract / can_reindex | 生命周期管理 |
| 复核员 | 查看 Review Cockpit 中的工单状态 | 复核追溯 |

### 2.2 前置条件

- 已配置有效 token；
- 文档库中存在至少一条已发布文档；
- 管理员账号用于验证生命周期操作。

### 2.3 可用性基线

默认遵循 `00-全局可用性基线与验收规则.md`。本模块额外强调：

- 生命周期操作是破坏性操作，必须有二次确认和原因输入；
- post-publish 的 chunk 编辑直接生效，必须明确告知用户影响；
- 文档状态徽标组必须准确反映当前生命周期状态。

## 3. 全链路场景叙事（A - 主结构）

### 场景 1：管理员查看并归档文档

**用户**：管理员
**目标**：查看已发布文档详情，并执行归档操作。
**优先级**：P0

#### 步骤 1：进入文档详情页

**操作描述**：
1. 管理员从 `/documents` 点击文件名；
2. 进入 `/documents/[docId]`。

**预期界面状态**：
- 页面加载时展示骨架屏；
- 数据加载完成后展示头部、三个 Tab、右侧边栏；
- 头部展示：
  - 文档状态徽标；
  - Review 状态 Badge；
  - Task 状态；
  - STALE 标记（如适用）；
  - 降级部件标记；
  - 四宫格指标（Findings、Blocking、Chunks、Draft edits）；
  - 元数据网格（Doc ID、Collection、Source File、Parse Snapshot）。

**[B] 单点验收点**：
- `DCD-001`：头部状态徽标组展示正确；
- `DCD-002`：STALE 标记在文档过期时显示；
- `DCD-003`：四宫格指标与元数据网格正确；
- `DCD-004`：文档标题回退链正确。

**[C] 视觉规格**：
- STALE 标记使用 destructive 红色 Badge，醒目；
- 状态徽标组间距一致；
- 四宫格卡片数字突出。

#### 步骤 2：查看 Source / Chunks / Agent Tab

**操作描述**：
1. 管理员切换三个 Tab，查看内容。

**预期界面状态**：
- Source Tab：与复核详情页 DocumentViewer 功能一致；
- Chunks Tab：与复核详情页 ChunkEditorWorkbench 功能一致，但为 post-publish 模式；
- Agent Tab：与复核详情页 AgentReviewPanel 功能一致。

**[B] 单点验收点**：
- `DCD-005`：Source Tab 正常预览；
- `DCD-006`：Chunks Tab 可查看/编辑 chunks；
- `DCD-007`：post-publish 编辑直接 Save 生效（Revision）；
- `DCD-008`：Agent Tab findings 按严重程度排序；
- `DCD-009`：三个 Tab 切换不丢失内容。

**[C] 视觉规格**：
- post-publish 编辑按钮使用蓝色主题（与 pre-publish 橙色区分）；
- Save 按钮文案明确。

#### 步骤 3：在 Review Cockpit 查看关联工单

**操作描述**：
1. 管理员查看右侧 Review Cockpit。

**预期界面状态**：
- 若文档关联了复核工单，展示工单状态、决策、决策原因；
- 待复核状态 + 有权限时，提供 Approve/Reject/Return 按钮 + 原因输入；
- 已决策时展示决策详情。

**[B] 单点验收点**：
- `DCD-010`：关联工单状态展示正确；
- `DCD-011`：待复核且有权限时可审批；
- `DCD-012`：决策原因展示或显示 “No additional reason”。

#### 步骤 4：执行归档操作

**操作描述**：
1. 管理员在右侧 Lifecycle Actions 中点击“Archive Document”；
2. 在确认弹窗中输入操作原因；
3. 点击 Confirm。

**预期界面状态**：
- Archive 按钮在管理员有 can_archive 权限时可用；
- 点击后弹出确认 Dialog；
- 输入原因后确认；
- 操作成功后 Toast 提示；
- 文档状态更新为 Archived；
- 无权限时展示提示文案。

**[B] 单点验收点**：
- `DCD-013`：Archive 按钮权限控制正确；
- `DCD-014`：归档操作需确认并输入原因；
- `DCD-015`：归档后状态更新；
- `DCD-016`：无权限时展示提示。

### 场景 2：操作员分享与导出文档

**用户**：操作员
**目标**：分享文档链接或导出 JSON。
**优先级**：P1

#### 步骤 1：分享文档

**操作描述**：
1. 操作员点击“分享”按钮；
2. 在分享弹窗中复制链接；
3. 关闭弹窗。

**预期界面状态**：
- 分享弹窗展示文档链接；
- 提供复制按钮；
- 复制后有成功反馈。

**[B] 单点验收点**：
- `DCD-017`：分享弹窗可打开；
- `DCD-018`：链接可复制；
- `DCD-019`：复制后有反馈。

#### 步骤 2：导出 JSON

**操作描述**：
1. 操作员点击“导出 JSON”按钮；
2. 系统下载 JSON 文件。

**预期界面状态**：
- 导出按钮可点击；
- 浏览器发起下载；
- 文件内容包含文档投影数据。

**[B] 单点验收点**：
- `DCD-020`：导出 JSON 可用；
- `DCD-021`：下载文件内容正确。

### 场景 3：查看相关文档

**用户**：操作员
**目标**：在文档详情页查看相关文档推荐。
**优先级**：P2

**预期界面状态**：
- 详情页底部或侧边展示“相关文档”区域；
- 列出若干推荐文档；
- 点击可跳转对应详情页。

**[B] 单点验收点**：
- `DCD-022`：相关文档区域展示；
- `DCD-023`：点击可跳转。

## 4. 异常 / 容错场景

### 场景 4：文档不存在或越权访问

**优先级**：P0
**异常点**：用户访问了不存在的 docId 或无权限的文档。

**预期系统行为**：
- 展示 404 页面或权限错误提示；
- 提供返回文档库入口。

**验收点**：
- `DCD-ERR-001`：不存在 docId 展示 not-found；
- `DCD-ERR-002`：越权访问被拦截。

### 场景 5：生命周期操作失败

**优先级**：P1
**异常点**：Archive/Retract/Reindex API 失败。

**预期系统行为**：
- 操作失败后 Toast 提示错误原因；
- 文档状态不发生变化；
- 用户可以重试。

**验收点**：
- `DCD-ERR-003`：失败后有错误提示；
- `DCD-ERR-004`：状态不错误更新。

### 场景 6：无权限用户看到生命周期按钮

**优先级**：P0
**异常点**：非管理员看到 Archive/Retract/Reindex 按钮。

**预期系统行为**：
- 按钮不可见或 disabled；
- 展示“您无权限执行此操作”提示。

**验收点**：
- `DCD-ERR-005`：非管理员无法触发生命周期操作。

### 场景 7：文档已 STALE

**优先级**：P1
**异常点**：文档过期标记为 STALE。

**预期系统行为**：
- 头部显示红色 STALE Badge；
- 提示用户文档可能已过期；
- 提供重新索引或更新入口（如已实现）。

**验收点**：
- `DCD-ERR-006`：STALE 标记醒目；
- `DCD-ERR-007`：提示文案清晰。

## 6. 验收结论与判定规则

| 缺陷类型 | 判定规则 | 结论 |
|---|---|---|
| 功能可用缺陷 | Tab 无法切换、生命周期操作失败、无权限可操作 | 失败，阻塞发布 |
| 视觉缺陷 | STALE 标记/状态徽标样式错误 | 记录，P0 阻塞 |
| 流程链路缺陷 | 审批通过但文档未发布、归档后状态未更新 | 失败，阻塞发布 |
| 数据一致性缺陷 | 文档详情状态与 /documents 列表、/retrieval 结果不一致 | 失败，阻塞发布 |
