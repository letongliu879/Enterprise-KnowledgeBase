# 05 - 文档库验收

> **文档定位**：本文件验证文档库页（`/documents`）的列表展示、筛选、视图切换、批量操作、收藏/最近访问、快速预览等能力。文档库是已发布文档的生命周期管理入口。
>
> **涉及页面**：`/documents`。
>
> **关联需求**：`需求.md` 第 6 章、第 14.5 节。

---

## 1. 验收目标与范围

### 1.1 业务目标

验证用户能够：

1. 在文档库中查看所有已发布文档；
2. 通过关键词、集合、状态、文件类型等 7 个维度筛选文档；
3. 在列表/卡片/网格三种视图中切换；
4. 快速预览文档内容；
5. 收藏常用文档、查看最近访问；
6. 管理员执行批量归档/撤回/重建索引操作。

### 1.2 涉及页面

| 路由 | 页面 | 作用 |
|---|---|---|
| `/documents` | 文档库页 | 文档列表与管理 |
| `/documents/[docId]` | 文档详情页 | 单文档生命周期管理 |

### 1.3 涉及角色

- **操作员**：查看文档、搜索、收藏、快速预览；
- **管理员**：批量操作、查看 Admin lifecycle Badge、执行生命周期管理。

---

## 2. 用户角色、前置条件与可用性基线

### 2.1 角色与权限

| 角色 | 权限 | 本页面使用场景 |
|---|---|---|
| 操作员 | 查看文档列表、搜索、收藏、预览 | 主要用户 |
| 管理员 | 批量归档/撤回/重建索引 | 生命周期管理 |

### 2.2 前置条件

- 已配置有效 token；
- 文档库中存在至少一条已发布文档（投影数据）；
- 管理员账号用于验证批量操作。

### 2.3 可用性基线

默认遵循 `00-全局可用性基线与验收规则.md`。本模块额外强调：

- 文档列表必须能处理最多 200 条投影数据而不卡顿；
- 批量操作必须明确展示影响范围，防止误操作；
- 筛选器组合必须稳定，不能互相冲突导致无结果。

## 3. 全链路场景叙事（A - 主结构）

### 场景 1：操作员查找并预览文档

**用户**：操作员
**目标**：在文档库中快速找到目标文档并预览。
**优先级**：P0

#### 步骤 1：进入文档库

**操作描述**：
1. 操作员点击侧边栏“文档库”进入 `/documents`。

**预期界面状态**：
- 页面加载时展示骨架屏；
- 数据加载完成后展示文档列表；
- 每条文档卡片显示：
  - 复选框（管理员可见/可用）；
  - 文件类型图标（PDF/DOC/PPT/Sheet，按扩展名匹配不同颜色）；
  - 文件名（可点击跳转 `/documents/[docId]`）；
  - Review 状态 Badge；
  - Indexed Badge；
  - 集合 ID；
  - chunk 数；
  - 页数；
  - 完整度（Complete/Partial）；
  - 相对更新时间；
  - 降级原因（如有）；
  - 文档状态徽标（Active/Pending/Archived）；
  - Stale 标记（红色 destructive Badge）；
  - 右侧 Chevron 跳转按钮。

**[B] 单点验收点**：
- `DOC-001`：文档按 `projection_updated_at` 倒序展示，最多 200 条；
- `DOC-002`：文件类型图标按扩展名区分颜色；
- `DOC-003`：文件名点击跳转 `/documents/[docId]`；
- `DOC-004`：Review / Indexed / 文档状态 / Stale 徽标正确；
- `DOC-005`：管理员可见复选框。

**[C] 视觉规格**：
- 文件类型图标颜色：PDF 红色、DOC 蓝色、PPT 橙色、Sheet 绿色；
- Stale 标记使用 destructive 红色 Badge；
- Indexed Badge 使用 success 绿色。

#### 步骤 2：使用筛选器查找文档

**操作描述**：
1. 操作员在筛选区输入关键词；
2. 选择集合、状态、复核状态、文件类型、新鲜度、索引状态；
3. 观察筛选结果；
4. 点击“清除筛选”。

**预期界面状态**：
- 筛选器包含 7 个维度：
  - 关键词搜索（文件名或 doc_id 模糊匹配）；
  - 集合筛选；
  - 状态筛选（Active/Pending/Archived）；
  - 复核状态筛选（Pending/Approved/Rejected）；
  - 文件类型筛选（PDF/Document/Presentation/Spreadsheet）；
  - 新鲜度筛选（Stale only / Fresh only）；
  - 索引状态筛选（Indexed / Not indexed）。
- 可见文档计数展示；
- 筛选结果即时更新；
- 清除筛选按钮在非默认时展示。

**[B] 单点验收点**：
- `DOC-006`：7 个筛选器均可用；
- `DOC-007`：关键词支持文件名和 doc_id 模糊匹配；
- `DOC-008`：组合筛选结果正确；
- `DOC-009`：可见文档计数准确；
- `DOC-010`：清除筛选恢复默认。

**[C] 视觉规格**：
- 筛选器在列表上方水平或折叠排列；
- 当前激活的筛选器有视觉指示；
- 可见计数使用 Badge 或文本清晰展示。

#### 步骤 3：切换视图

**操作描述**：
1. 操作员点击视图切换按钮：列表 / 卡片 / 网格。

**预期界面状态**：
- 列表视图：行式展示，信息密度高；
- 卡片视图：卡片网格，信息适中；
- 网格视图：更紧凑的网格布局；
- 切换时数据保持一致。

**[B] 单点验收点**：
- `DOC-011`：三种视图可切换；
- `DOC-012`：切换后数据一致；
- `DOC-013`：当前视图有高亮指示。

#### 步骤 4：快速预览文档

**操作描述**：
1. 操作员点击文档卡片上的预览按钮或文件名旁的眼睛图标；
2. DocumentPreviewDialog 浮层打开；
3. 查看文档基本信息和预览内容；
4. 关闭浮层。

**预期界面状态**：
- 浮层以 Dialog 形式展示；
- 展示文档标题、状态、元数据、内容预览；
- 可关闭。

**[B] 单点验收点**：
- `DOC-014`：预览按钮可点击；
- `DOC-015`：浮层展示文档基本信息；
- `DOC-016`：浮层可关闭。

#### 步骤 5：收藏与最近访问

**操作描述**：
1. 操作员点击文档卡片上的收藏按钮（星形图标）；
2. 查看收藏文档是否置顶或在收藏区展示；
3. 点击进入文档详情后再返回；
4. 查看最近访问区域。

**预期界面状态**：
- 收藏后星形图标变为实心/高亮；
- 收藏文档在列表顶部或单独收藏区展示；
- 最近访问的文档在快捷入口展示；
- 最近访问按时间倒序。

**[B] 单点验收点**：
- `DOC-017`：收藏按钮可切换状态；
- `DOC-018`：收藏文档置顶/单独展示；
- `DOC-019`：最近访问按时间倒序；
- `DOC-020`：最近访问可点击进入详情。

### 场景 2：管理员批量操作

**用户**：管理员
**目标**：对多个文档执行归档/撤回/重建索引。
**优先级**：P1

#### 步骤 1：选择文档

**操作描述**：
1. 管理员勾选多个文档的复选框；
2. 或使用“全选当前页”。

**预期界面状态**：
- 选择后底部/顶部出现批量操作栏；
- 操作栏显示已选数量；
- 非管理员看不到复选框或操作栏。

**[B] 单点验收点**：
- `DOC-021`：管理员可选择多个文档；
- `DOC-022`：全选/取消全选当前页可用；
- `DOC-023`：非管理员无批量操作入口。

#### 步骤 2：执行批量归档

**操作描述**：
1. 管理员点击“Archive（归档）”；
2. 输入操作原因；
3. 确认执行。

**预期界面状态**：
- 确认 Dialog 展示影响文档数量；
- 原因输入框必填或可选（按实现）；
- 操作成功后 Toast 提示；
- 列表中文档状态更新为 Archived。

**[B] 单点验收点**：
- `DOC-024`：批量归档需确认；
- `DOC-025`：归档后状态更新；
- `DOC-026`：操作结果展示成功/失败数。

#### 步骤 3：执行批量重建索引

**操作描述**：
1. 管理员点击“Reindex（重建索引）”；
2. 输入 `index_profile_id`；
3. 输入操作原因；
4. 确认执行。

**预期界面状态**：
- 确认 Dialog 要求输入 index_profile_id；
- 输入验证；
- 操作成功后状态或版本更新。

**[B] 单点验收点**：
- `DOC-027`：Reindex 需输入 index_profile_id；
- `DOC-028`：index_profile_id 有校验；
- `DOC-029`：操作结果明细展示。

## 4. 异常 / 容错场景

### 场景 3：文档库为空

**优先级**：P0
**异常点**：没有任何已发布文档。

**预期系统行为**：
- 展示 EmptyState；
- 提示用户去上传或等待文档入库；
- 提供“去上传”按钮。

**验收点**：
- `DOC-ERR-001`：空库展示 EmptyState；
- `DOC-ERR-002`：提供上传入口。

### 场景 4：筛选组合无结果

**优先级**：P0
**异常点**：7 个筛选器组合后无匹配文档。

**预期系统行为**：
- 展示“无匹配结果”提示；
- 提供“清除筛选”按钮；
- 保留筛选器状态。

**验收点**：
- `DOC-ERR-003`：无匹配提示清晰；
- `DOC-ERR-004`：清除筛选可恢复。

### 场景 5：非管理员尝试批量操作

**优先级**：P0
**异常点**：普通用户看到批量操作入口。

**预期系统行为**：
- 非管理员看不到复选框；
- 或复选框 disabled；
- 批量操作栏不展示。

**验收点**：
- `DOC-ERR-005`：非管理员无批量操作入口。

### 场景 6：批量操作失败

**优先级**：P1
**异常点**：批量归档/撤回/Reindex 部分失败。

**预期系统行为**：
- 操作结果展示总数量、成功数、失败数；
- 失败项给出原因；
- 成功的项状态已更新。

**验收点**：
- `DOC-ERR-006`：批量操作结果明细完整；
- `DOC-ERR-007`：失败原因清晰。

## 6. 验收结论与判定规则

| 缺陷类型 | 判定规则 | 结论 |
|---|---|---|
| 功能可用缺陷 | 列表不加载、筛选无效、跳转错误、批量操作失败 | 失败，阻塞发布 |
| 视觉缺陷 | 图标颜色/Badge/视图切换样式异常 | 记录，P0 阻塞 |
| 流程链路缺陷 | 审批通过文档未出现在文档库、状态不同步 | 失败，阻塞发布 |
| 数据一致性缺陷 | 文档库状态与 /documents/[docId]、/retrieval 结果不一致 | 失败，阻塞发布 |
