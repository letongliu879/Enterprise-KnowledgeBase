# 07 - 检索验证验收

> **文档定位**：本文件验证检索验证页（`/retrieval`）的检索表单、结果展示、证据片段、调试信息、历史记录、结果对比等能力。该页面是验证知识库检索效果的核心工具。
>
> **涉及页面**：`/retrieval`、`/retrieval/profiles`（关联）。
>
> **关联需求**：`需求.md` 第 8 章、第 13.3 节、第 14.7 节。

---

## 1. 验收目标与范围

### 1.1 业务目标

验证用户能够：

1. 配置查询、Token 预算、集合、检索配置等参数；
2. 执行检索并查看证据片段列表；
3. 理解每条证据的相关性分数、元数据和选中原因；
4. 复制证据内容、导出结果、对比不同配置；
5. 在空结果时获得明确提示；
6. 管理检索配置（通过 `/retrieval/profiles`）。

### 1.2 涉及页面

| 路由 | 页面 | 作用 |
|---|---|---|
| `/retrieval` | 检索验证页 | 执行检索、查看证据片段 |
| `/retrieval/profiles` | 检索配置管理 | CRUD + 发布检索配置 |

### 1.3 涉及角色

- **操作员**：执行检索、查看结果、导出；
- **管理员**：管理检索配置。

---

## 2. 用户角色、前置条件与可用性基线

### 2.1 角色与权限

| 角色 | 权限 | 本页面使用场景 |
|---|---|---|
| 操作员 | 执行检索、复制/导出结果、查看历史 | 主要用户 |
| 管理员 | 管理 retrieval profiles | 配置管理 |

### 2.2 前置条件

- 已配置有效 token；
- 已选择知识库集合；
- 已存在至少一个已发布的检索配置（retrieval profile）；
- 集合中已有索引的文档，确保能检索到结果。

### 2.3 可用性基线

默认遵循 `00-全局可用性基线与验收规则.md`。本模块额外强调：

- 检索是异步操作，必须有 loading 反馈；
- 证据片段的分数和元数据必须清晰可见；
- 未选择集合/配置时必须明确禁用检索，避免无效请求。

## 3. 全链路场景叙事（A - 主结构）

### 场景 1：标准检索验证流程

**用户**：操作员
**目标**：执行一次检索并查看证据片段。
**优先级**：P0

#### 步骤 1：进入检索页并配置参数

**操作描述**：
1. 操作员点击侧边栏“检索验证”进入 `/retrieval`；
2. 在查询输入框输入查询词；
3. 查看/调整 Token 预算（默认 2000）；
4. 选择知识库集合；
5. 选择检索配置（retrieval profile）；
6. 选择调试级别（无 / 基础 / 完整）。

**预期界面状态**：
- 页面加载时展示表单和可能的检索历史；
- 查询输入框为标准检索字段，Enter 可触发检索；
- Token 预算为数字输入，默认 2000；
- 集合选择器下拉展示可选集合；
- 检索配置选择器展示已发布的 profiles；
- 调试级别选择有 Tooltip 解释各级含义；
- 未选择集合或配置时，检索按钮 disabled；
- 未选集合/配置时展示 Alert 提示。

**[B] 单点验收点**：
- `RET-001`：查询输入框 Enter 触发检索；
- `RET-002`：Token 预算默认 2000，可修改；
- `RET-003`：集合选择器加载可选集合；
- `RET-004`：检索配置选择器加载已发布 profiles；
- `RET-005`：调试级别 Tooltip 解释清晰；
- `RET-006`：未选集合/配置时检索按钮 disabled 并提示。

**[C] 视觉规格**：
- 检索表单区域布局清晰，输入框宽度充足；
- 禁用状态的检索按钮视觉明显；
- Alert 提示位于表单上方或按钮附近。

#### 步骤 2：执行检索

**操作描述**：
1. 操作员点击“检索”按钮；
2. 等待结果返回。

**预期界面状态**：
- 点击检索按钮后按钮进入 loading 状态；
- 结果显示区域展示 Skeleton 或加载提示；
- 检索成功后展示证据片段列表；
- 检索耗时显示在结果头部。

**[B] 单点验收点**：
- `RET-007`：检索按钮 loading 状态；
- `RET-008`：结果返回前展示加载态；
- `RET-009`：检索成功后展示 evidence_items；
- `RET-010`：检索耗时正确显示。

#### 步骤 3：查看证据片段

**操作描述**：
1. 操作员查看结果头部信息；
2. 展开/折叠证据卡片；
3. 查看相关性分数和元数据；
4. 复制证据内容。

**预期界面状态**：
- 结果头部：标题 + 展开全部/收起全部按钮 + 计数 Badge（X 项 · 已用 Y Token）；
- 每条证据卡片：
  - 序号 Badge（渐变背景）；
  - 相关性分数（Score）+ 可视化进度条；
  - 复制内容按钮（带 1.5s 成功状态反馈）；
  - 展开/折叠按钮；
  - 元数据行：doc_id、evidence_id、collection_id；
  - 内容区（默认折叠 4 行）；
  - 展开后展示：source_stage、why_selected、section_path。

**[B] 单点验收点**：
- `RET-011`：结果头部展示项数和 Token 使用量；
- `RET-012`：每条证据有序号 Badge、score、进度条；
- `RET-013`：复制按钮有 1.5s 成功反馈；
- `RET-014`：展开后展示 source_stage/why_selected/section_path；
- `RET-015`：展开全部/收起全部按钮生效。

**[C] 视觉规格**：
- 序号 Badge 使用渐变背景；
- 相关性分数进度条颜色随分数变化（高分绿色、低分黄色/红色）；
- 证据卡片间距一致，展开后内容区有清晰分隔。

### 场景 2：检索结果对比

**用户**：操作员/管理员
**目标**：使用两套配置并行检索并对比结果。
**优先级**：P1

#### 步骤 1：启用结果对比

**操作描述**：
1. 用户在检索表单中选择“对比模式”；
2. 选择两套检索配置；
3. 执行检索。

**预期界面状态**：
- 检索结果区域分为左右两栏；
- 左侧展示配置 A 的结果，右侧展示配置 B 的结果；
- 相同/不同证据有高亮或标记。

**[B] 单点验收点**：
- `RET-016`：对比模式可启用；
- `RET-017`：两套配置结果并行展示；
- `RET-018`：对比结果清晰可辨。

### 场景 3：检索结果反馈

**用户**：操作员
**目标**：对检索结果点赞/点踩以收集反馈。
**优先级**：P2

**预期界面状态**：
- 每条证据卡片或结果头部有 👍/👎 按钮；
- 点击后有视觉反馈；
- 反馈可被记录（通过 API）。

**[B] 单点验收点**：
- `RET-019`：👍/👎 按钮可点击；
- `RET-020`：点击后有反馈动画或状态变化。

### 场景 4：导出检索结果

**用户**：操作员
**目标**：将检索结果导出为 JSON 或 Markdown。
**优先级**：P1

**预期界面状态**：
- 结果区域有导出按钮；
- 支持 JSON / Markdown 格式；
- 导出文件内容完整。

**[B] 单点验收点**：
- `RET-021`：导出按钮可用；
- `RET-022`：支持 JSON / Markdown；
- `RET-023`：导出内容正确。

## 4. 异常 / 容错场景

### 场景 5：检索无结果

**优先级**：P0
**异常点**：查询词无匹配证据。

**预期系统行为**：
- 展示 EmptyState：“无证据片段”；
- 提示检查查询词、集合、配置；
- 展示热门查询、最近查询、示例查询（空查询提示）。

**验收点**：
- `RET-ERR-001`：无结果展示 EmptyState；
- `RET-ERR-002`：提供查询建议；
- `RET-ERR-003`：页面不报错。

### 场景 6：未选集合/配置执行检索

**优先级**：P0
**异常点**：用户未选择集合或配置就点击检索。

**预期系统行为**：
- 检索按钮 disabled；
- 展示 Alert 提示“请选择集合和检索配置”。

**验收点**：
- `RET-ERR-004`：未选时按钮禁用；
- `RET-ERR-005`：提示文案准确。

### 场景 7：检索 API 失败

**优先级**：P0
**异常点**：`/workbench/retrieve` 返回错误。

**预期系统行为**：
- 展示错误 Alert 或 Toast；
- 不展示旧结果或虚假结果；
- 提供重试入口。

**验收点**：
- `RET-ERR-006`：API 失败有错误提示；
- `RET-ERR-007`：不展示过期结果。

### 场景 8：检索历史回填

**优先级**：P1
**异常点**：用户想复用之前的查询。

**预期系统行为**：
- 展示最近查询历史（localStorage）；
- 点击历史项一键回填查询和参数；
- 热门查询和示例查询帮助用户开始。

**验收点**：
- `RET-ERR-008`：最近查询可回填；
- `RET-ERR-009`：示例查询可点击执行。

## 6. 验收结论与判定规则

| 缺陷类型 | 判定规则 | 结论 |
|---|---|---|
| 功能可用缺陷 | 无法检索、结果不展示、复制/导出失败 | 失败，阻塞发布 |
| 视觉缺陷 | 证据卡片/进度条/Badge 样式异常 | 记录，P0 阻塞 |
| 流程链路缺陷 | 审批发布文档后检索不到、Token 显示错误 | 失败，阻塞发布 |
| 数据一致性缺陷 | /retrieval 检索结果中的 doc_id/evidence_id 在 /documents 中不存在或状态不一致 | 失败，阻塞发布 |
