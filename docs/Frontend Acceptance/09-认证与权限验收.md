# 09 - 认证与权限验收

> **文档定位**：本文件验证设置页（`/settings`）最核心的认证配置和权限范围编辑能力。认证（JWT/API Key）决定用户能否调用后端，权限范围（accessScope）决定上传等操作的权限校验上下文。
>
> **涉及页面**：`/settings`。
>
> **关联需求**：`需求.md` 第 10.1 节、第 10.2 节、第 10.3 节、第 1.2 节（全局状态）。

---

## 1. 验收目标与范围

### 1.1 业务目标

验证用户能够：

1. 配置演示 JWT 令牌和 API 密钥；
2. 配置权限范围（内部/外部模式）；
3. 所有配置变更保存到 Zustand 并持久化到 localStorage；
4. 配置正确影响其他页面的前置校验（如 `/upload` 的权限范围校验）。

### 1.2 涉及页面

| 路由 | 页面 | 作用 |
|---|---|---|
| `/settings` | 设置首页 | 认证 + 权限范围 |

### 1.3 涉及角色

- **操作员**：配置 token、权限范围；
- **所有用户**：首次使用必须完成认证配置。

---

## 2. 用户角色、前置条件与可用性基线

### 2.1 角色与权限

| 角色 | 权限 | 本页面使用场景 |
|---|---|---|
| 所有用户 | 配置自己的 token/scope | 认证与权限 |

### 2.2 前置条件

- 应用可访问；
- 用户处于演示/操作员认证模式。

### 2.3 可用性基线

默认遵循 `00-全局可用性基线与验收规则.md`。本模块额外强调：

- Token/API Key 是敏感信息，输入框必须为 password 类型；
- 设置变更必须明确保存，不能静默生效或丢失；
- Tabs 切换必须有动画指示器，当前 Tab 状态清晰。

---

## 3. 全链路场景叙事（A - 主结构）

### 场景 1：用户配置认证信息

**用户**：操作员
**目标**：在设置页配置 JWT 和 API Key，使其他页面能正常调用后端。
**优先级**：P0

#### 步骤 1：进入设置页并切换认证 Tab

**操作描述**：
1. 用户点击 Header 设置入口或侧边栏相关入口进入 `/settings`；
2. 页面默认展示认证 Tab 或权限范围 Tab；
3. 点击“认证”Tab。

**预期界面状态**：
- 设置页使用自定义 Tabs 切换；
- Tab 切换带 layoutId 弹簧动画的活跃指示器；
- 认证 Tab 展示两个 password 类型输入框：演示 JWT 令牌、演示 API 密钥；
- 保存按钮初始状态取决于当前值是否与 store 一致。

**[B] 单点验收点**：
- `AUTH-001`：Tabs 切换有动画指示器；
- `AUTH-002`：当前 Tab 状态清晰；
- `AUTH-003`：认证 Tab 展示 JWT 和 API Key 输入框；
- `AUTH-004`：输入框为 password 类型。

**[C] 视觉规格**：
- Tabs 位于页面顶部或左侧；
- 活跃指示器使用主题主色，带弹簧动画；
- 输入框有眼睛图标可切换显示/隐藏。

#### 步骤 2：输入并保存凭证

**操作描述**：
1. 用户粘贴 JWT 令牌到“演示 JWT 令牌”输入框；
2. 粘贴 API 密钥到“演示 API 密钥”输入框；
3. 点击保存按钮。

**预期界面状态**：
- 保存按钮更新 Zustand store；
- Toast 提示保存成功；
- 刷新页面后配置从 localStorage 恢复；
- 配置生效后其他页面（如上传、检索）可正常调用 API。

**[B] 单点验收点**：
- `AUTH-005`：保存后 Zustand 更新；
- `AUTH-006`：保存后 Toast 成功；
- `AUTH-007`：刷新后配置持久化；
- `AUTH-008`：配置生效于其他页面。

### 场景 2：用户配置权限范围

**用户**：操作员
**目标**：配置 accessScope，用于上传等操作的权限校验。
**优先级**：P0

#### 步骤 1：切换权限范围 Tab

**操作描述**：
1. 用户点击“权限范围”Tab。

**预期界面状态**：
- 权限范围 Tab 激活；
- 展示范围类型切换：内部（Internal）/ 外部（External），Segmented Control 样式。

**[B] 单点验收点**：
- `AUTH-009`：权限范围 Tab 可切换；
- `AUTH-010`：范围类型切换为 Segmented Control。

#### 步骤 2：配置内部模式

**操作描述**：
1. 用户选择“内部”模式；
2. 填写部门、角色、用户、组字段；
3. 点击保存。

**预期界面状态**：
- 内部模式字段：部门、角色、用户、组；
- 保存后组装 scope 对象存入 Zustand；
- Toast 提示成功；
- 权限范围用于 `/upload` 等页面的前置校验。

**[B] 单点验收点**：
- `AUTH-011`：内部模式字段正确；
- `AUTH-012`：保存后 scope 存入 Zustand；
- `AUTH-013`：上传页能读取 scope 做校验。

#### 步骤 3：配置外部模式

**操作描述**：
1. 用户选择“外部”模式；
2. 填写代理类型 ID、API 密钥、客户、应用字段；
3. 点击保存。

**预期界面状态**：
- 外部模式字段：代理类型 ID、API 密钥、客户、应用；
- 保存后 scope 对象更新。

**[B] 单点验收点**：
- `AUTH-014`：外部模式字段正确；
- `AUTH-015`：切换模式时表单字段正确切换。

## 4. 异常 / 容错场景

### 场景 3：保存失败

**优先级**：P0
**异常点**：网络中断或后端错误导致设置保存失败。

**预期系统行为**：
- Toast 提示保存失败；
- 不更新 Zustand；
- 用户输入保留。

**验收点**：
- `AUTH-ERR-001`：保存失败有错误提示；
- `AUTH-ERR-002`：不覆盖原有配置。

### 场景 4：Token 无效导致其他页面失败

**优先级**：P0
**异常点**：用户配置了错误 token。

**预期系统行为**：
- 其他页面 API 401；
- 页面提示重新配置 token；
- 设置页可重新修改。

**验收点**：
- `AUTH-ERR-003`：401 后引导回设置页；
- `AUTH-ERR-004`：设置页可修正。

### 场景 5：权限范围未配置导致上传拦截

**优先级**：P0
**异常点**：用户未配置 accessScope 就尝试上传。

**预期系统行为**：
- `/upload` 上传区禁用；
- 提示配置权限范围。

**验收点**：
- `AUTH-ERR-005`：未配置 scope 时上传被拦截。

### 场景 6：Token/API Key 输入框未隐藏

**优先级**：P0
**异常点**：敏感信息明文显示且无切换能力。

**预期系统行为**：
- 输入框默认 password 类型；
- 提供眼睛图标切换显示/隐藏。

**验收点**：
- `AUTH-ERR-006`：默认隐藏敏感信息；
- `AUTH-ERR-007`：可切换显示。

## 6. 验收结论与判定规则

| 缺陷类型 | 判定规则 | 结论 |
|---|---|---|
| 功能可用缺陷 | token 无法保存、权限范围不生效、输入框不隐藏 | 失败，阻塞发布 |
| 视觉缺陷 | Tabs 动画/输入框样式异常 | 记录，P0 阻塞 |
| 数据一致性缺陷 | 设置后上传/检索仍提示未配置 | 失败，阻塞发布 |
| 流程链路缺陷 | 认证配置未同步到 API 客户端 | 失败，阻塞发布 |

## 7. 已知缺口与未实现功能

本模块聚焦认证配置（JWT/API Key）和权限范围（accessScope）。根据 `需求.md` 第 10.2、10.3 节及第 14.9 节，认证与权限部分无标注为 ❌ 未实现 或 ⏳ 部分实现 的独立缺口。

系统管理相关功能（个人资料、偏好设置、安全设置、数据管理、通知偏好、快捷键、账户管理、关于页面、API 密钥、审计日志、解析策略）的验收与缺口详见 `09b-系统管理验收.md`。
