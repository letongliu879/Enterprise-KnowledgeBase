# 03 - 人工复核队列验收

> **文档定位**：本文件验证人工复核队列页（`/review`）的列表展示、筛选、排序、分页、批量操作、空状态及队列统计等能力。复核队列是操作员和复核员日常最高频的入口之一。
>
> **涉及页面**：`/review`。
>
> **关联需求**：`需求.md` 第 4 章、第 14.3 节。

---

## 1. 验收目标与范围

### 1.1 业务目标

验证用户能够：

1. 在队列中快速找到需要复核的工单；
2. 通过筛选、排序、搜索缩小范围；
3. 识别工单的紧急程度和状态；
4. 点击进入详情页进行审批；
5. 批量处理多个工单（如已实现）；
6. 在无工单时获得正向反馈。

### 1.2 涉及页面

| 路由 | 页面 | 作用 |
|---|---|---|
| `/review` | 人工复核队列页 | 复核工单列表、筛选、批量操作 |
| `/review/[taskId]` | 复核详情页 | 单条工单审批 |

### 1.3 涉及角色

- **复核员**：查看待复核工单并审批；
- **操作员**：查看自己上传文件生成的工单状态；
- **管理员**：批量批准/拒绝、查看全部工单。

---

## 2. 用户角色、前置条件与可用性基线

### 2.1 角色与权限

| 角色 | 权限 | 本页面使用场景 |
|---|---|---|
| 复核员 | 查看工单、进入详情、审批 | 主要用户 |
| 操作员 | 查看自己相关的工单 | 追踪上传结果 |
| 管理员 | 批量操作、查看全量 | 队列管理 |

### 2.2 前置条件

- 已配置有效 token；
- 已选择知识库集合；
- 队列中存在至少一条工单（待复核/已批准/已拒绝/已退回）。

### 2.3 可用性基线

默认遵循 `00-全局可用性基线与验收规则.md`。本模块额外强调：

- 队列必须实时反映最新状态（自动轮询）；
- 筛选和搜索必须即时反馈；
- 批量操作必须有二次确认，防止误操作。

## 3. 全链路场景叙事（A - 主结构）

### 场景 1：复核员处理待复核工单

**用户**：复核员
**目标**：在队列中找到待复核工单并进入详情审批。
**优先级**：P0

#### 步骤 1：进入复核队列

**操作描述**：
1. 复核员点击侧边栏“人工复核”进入 `/review`。

**预期界面状态**：
- 页面加载时展示骨架屏；
- 数据加载完成后展示工单卡片列表；
- 每条工单卡片显示：
  - 左侧状态色条；
  - 状态图标（待复核/已批准/已拒绝/已退回）；
  - 文件名（回退到标题/doc_id/ticket_id）；
  - 集合 ID Badge；
  - 工单 ID（monospace）；
  - 相对时间（刚刚/N分钟前/N天前）；
  - 状态徽标（待复核带脉冲动画）；
  - 右侧 Chevron 导航指示。
- 顶部有队列统计栏和筛选器。

**[B] 单点验收点**：
- `REV-001`：工单按更新时间倒序展示；
- `REV-002`：卡片展示状态色条、状态图标、文件名、集合 ID、工单 ID、相对时间；
- `REV-003`：待复核状态徽标带脉冲动画；
- `REV-004`：点击卡片跳转 `/review/[taskId]`。

**[C] 视觉规格**：
- 工单卡片间距一致，hover 时有轻微阴影/位移；
- 工单 ID 使用 Geist Mono 等宽字体；
- 待复核脉冲动画柔和，不刺眼。

#### 步骤 2：使用筛选器缩小范围

**操作描述**：
1. 复核员在顶部筛选器中选择特定集合；
2. 选择状态“待复核”；
3. 在搜索框输入工单 ID 或文件名；
4. 点击“清除筛选”。

**预期界面状态**：
- 集合筛选下拉展示全部 + 可选集合；
- 状态筛选下拉展示全部、待复核、已批准、已拒绝、已退回；
- 筛选后列表即时刷新；
- 非默认筛选时显示“清除筛选”按钮；
- 搜索支持按工单 ID/文件名/文档 ID 模糊匹配。

**[B] 单点验收点**：
- `REV-005`：集合筛选过滤结果正确；
- `REV-006`：状态筛选过滤结果正确；
- `REV-007`：搜索框按工单 ID/文件名/文档 ID 搜索；
- `REV-008`：清除筛选按钮仅在非默认时展示；
- `REV-009`：清除筛选后恢复默认列表。

**[C] 视觉规格**：
- 筛选器在列表上方水平排列，间距一致；
- 清除筛选按钮为文字按钮或 subtle 样式；
- 筛选标签/徽标清晰显示当前筛选状态。

#### 步骤 3：查看队列统计

**操作描述**：
1. 复核员查看队列顶部的统计栏。

**预期界面状态**：
- 统计栏展示队列关键指标，如待复核数、已批准数、已拒绝数、已退回数、总数等；
- 数字随筛选结果或状态变化更新。

**[B] 单点验收点**：
- `REV-010`：队列统计栏展示关键指标；
- `REV-011`：统计数字与列表数据一致。

### 场景 2：批量审批工单

**用户**：管理员/复核员
**目标**：批量批准或拒绝多个工单。
**优先级**：P1

#### 步骤 1：选择多个工单

**操作描述**：
1. 用户在工单卡片上勾选复选框（如已实现批量选择）；
2. 或列表支持 Shift 多选；
3. 选择后底部/顶部出现批量操作栏。

**预期界面状态**：
- 选择工单后展示批量操作栏；
- 批量操作栏显示已选数量；
- 提供“批量批准”“批量拒绝”按钮；
- 提供全选/取消全选当前页。

**[B] 单点验收点**：
- `REV-012`：可选择多个工单；
- `REV-013`：批量操作栏在选择后出现；
- `REV-014`：全选/取消全选当前页可用。

#### 步骤 2：执行批量审批

**操作描述**：
1. 用户点击“批量批准”或“批量拒绝”；
2. 系统弹出确认 Dialog；
3. 用户输入原因（可选/必填）；
4. 确认后执行批量操作；
5. 查看操作结果。

**预期界面状态**：
- 确认 Dialog 展示受影响工单数量；
- 操作成功后 Toast 提示；
- 列表状态更新；
- 操作结果展示总数量、成功数、失败数及每条结果状态。

**[B] 单点验收点**：
- `REV-015`：批量批准成功更新工单状态；
- `REV-016`：批量拒绝成功更新工单状态；
- `REV-017`：操作结果展示成功/失败明细；
- `REV-018`：失败项给出原因。

## 4. 异常 / 容错场景

### 场景 3：队列为空

**优先级**：P0
**异常点**：当前没有任何复核工单。

**预期系统行为**：
- 展示 EmptyState（插图 + 标题 + 描述）；
- 文案正向，如“恭喜，队列已清空”；
- 提供“去上传”等可选操作按钮。

**验收点**：
- `REV-ERR-001`：空队列展示 EmptyState；
- `REV-ERR-002`：空状态文案正向友好。

### 场景 4：筛选后无结果

**优先级**：P0
**异常点**：筛选条件过于严格导致无匹配工单。

**预期系统行为**：
- 展示“无匹配结果”提示；
- 提供“清除筛选”按钮；
- 保留筛选器状态，方便用户调整。

**验收点**：
- `REV-ERR-003`：无匹配结果提示清晰；
- `REV-ERR-004`：清除筛选可恢复。

### 场景 5：自动刷新失败

**优先级**：P1
**异常点**：轮询更新时后端离线或返回错误。

**预期系统行为**：
- 不破坏当前列表；
- 展示离线或错误提示；
- 后端恢复后继续轮询。

**验收点**：
- `REV-ERR-005`：轮询失败不白屏；
- `REV-ERR-006`：恢复后继续轮询。

### 场景 6：无权限查看工单

**优先级**：P0
**异常点**：非管理员用户尝试查看不属于自己/自己部门的工单。

**预期系统行为**：
- 列表中不展示无权限工单；
- 或直接访问 `/review/[taskId]` 时返回 403/404；
- 展示 EmptyState 或错误提示。

**验收点**：
- `REV-ERR-007`：无权限工单不可见；
- `REV-ERR-008`：越权访问详情页被拦截。

## 6. 验收结论与判定规则

| 缺陷类型 | 判定规则 | 结论 |
|---|---|---|
| 功能可用缺陷 | 无法进入详情、筛选/搜索无效、批量操作失败 | 失败，阻塞发布 |
| 视觉缺陷 | 卡片/徽标/筛选器样式异常 | 记录，P0 阻塞 |
| 流程链路缺陷 | 审批后队列状态未更新、跳转详情 ID 错误 | 失败，阻塞发布 |
| 数据一致性缺陷 | /review 工单状态与 /review/[taskId]、/documents 状态不一致 | 失败，阻塞发布 |
